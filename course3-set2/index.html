Q1

リアクティブ Web アプリで『保存』ボタンがサーバーアクションを呼びます。実行中に処理中であることを視覚的に伝える、OutSystems のデフォルトの挙動は？

※画面遷移時はトップバーが出ますが、画面内のボタンから呼ぶ（ナビゲーションを伴わない）サーバーアクションに対する汎用の自動スピナーは既定では保証されず、開発者が独自に用意することが多い点に注意してください。

  • Aボタンのテキストが自動的に『Loading...』に変わる。
  • B画面が自動的にロックされグレーアウトされる。
  • C手動でローカル変数 IsLoading を切り替え、スピナーにバインドする必要がある。
  • Dサーバー呼び出し中、画面上部にロードバー（プログレスバー）を自動表示する。

正解：D

Reactive は画面遷移（ナビゲーション）時に画面上部の青いプログレスバーを自動表示します。

Q2

100通のメールに for each でメール送信中、50通目で例外が発生し、ループ内に例外ハンドラーがある場合どうなりますか？

  • A残りの50通が黙って送信される。
  • Bアプリケーションがクラッシュする。
  • Cループが停止し、トランザクションがロールバックされる。
  • D例外がキャッチされハンドラーのロジック（例：エラーログ）が実行され、その後ループは51通目に進む（継続可能に構成されている場合）。

正解：D

ループ内で例外を捕捉するハンドラー（またはラッパーアクション）があれば、1通の失敗でループ全体を止めず『エラー継続』パターンを実装できます。

Q3

フォームで『SubscribeToNewsletter』がオンのときだけ『Email』を必須にしたい。条件付き検証の実装は？

  • AEmail の Mandatory を False にし、保存画面アクション内で条件を確認し、満たさない場合 Input.Valid = False を設定する。
  • Bチェックボックスの On Change で Email の Mandatory を切り替える。
  • CEmail の Mandatory をブール式 SubscribeToNewsletter に設定する。
  • Dシナリオごとに別々の保存ボタンを作る。

正解：A

複雑な条件付き検証は、標準の Mandatory プロパティだけでは実現しにくく、保存アクション内で条件を確認し、空なら Input.Valid=False とし ValidationMessage を表示するのが定石です。

Q4

『4 Layer Canvas』アーキテクチャで、汎用の『SendEmail』ラッパーアクションはどこに配置すべき？

  • Aオーケストレーション層
  • Bエンドユーザ層
  • CFoundation 層（ライブラリ）
  • Dコア層

正解：C

メール送信・監査・文字列操作などビジネスに依存しない技術的ユーティリティは Foundation 層（ライブラリ）に配置します。

Q5

DB に保存された PDF をユーザーがダウンロードできるようにします。リアクティブ Web でのダウンロードロジックの実装は？

  • Aサーバーアクション内で 'Download' ノードを使用する。
  • B画面アクション（クライアント側）のロジックフロー内で 'Download' ノードを使用する。
  • Cリアクティブ Web ではファイルをダウンロードできない。
  • DDB を指す 'External Site' ノードを使用する。

正解：B

Reactive では 'Download' はクライアント（画面）アクションのロジックフロー内で使えます。サーバーから取得したバイナリとファイル名を渡すと、ブラウザのネイティブなダウンロードが起動します。

Q6

クライアントアクションからトリガーされるサーバーアクションのデフォルトタイムアウトは？

  • A100秒
  • B60秒
  • C無制限
  • D10秒

正解：D

モジュールの Server Request Timeout は既定10秒。10秒以内に応答がないと Communication Exception が発生します（UI の応答性維持のため）。長時間処理はタイマー等にオフロードします。

Q7

SQL クエリの入力パラメータ 'SearchKeyword' を、SQL インジェクションを防いで使うには？

  • AOutSystems の SQL は構文に関係なく自動的にインジェクションに免疫がある。
  • B送信前に EncodeHtml でサニタイズする。
  • C'@SearchKeyword' 構文（クエリパラメータ）で使用する。
  • D'@SearchKeyword' を使うが Expand Inline を有効にする。

正解：C

@SearchKeyword のパラメータ化クエリでは、DB エンジンが入力をリテラル値として扱いコードとして解釈しません。Expand Inline はテキストを直接埋め込むためインジェクションの脆弱性になります。

Q8

自動生成される『Check<Role>』アクションの目的は？

  • Aユーザーからロールを削除する。
  • Bロールをユーザーに割り当てる。
  • C現在ログイン中のユーザーがその特定のロールを持つか確認する。
  • Dロールが DB に存在するか確認する。

正解：C

Check<RoleName>Role(UserId) 関数は、ユーザーがそのロールを持てば True を返します。ロジックでセキュリティをプログラム的に強制する主要手段です（削除は Revoke、付与は Grant）。

Q9

リアクティブ画面のローカル変数 'IsFormValid' が False に変わったとき、すぐに警告を表示したい。実装は？

  • A'Data Action' で変数を監視する。
  • B変数の変化だけではロジックをトリガーできない。変数を変更した場所で明示的に副作用のロジックを呼ぶ必要がある。
  • C'OnRender' で変数値をチェックする。
  • D変数を 'If' ウィジェットにバインドし、その中にメッセージを置く。

正解：B

O11 にはローカル変数のウォッチャー（OnVariableChanged 等）はありません。変数を変更した直後に ShowMessage 等のロジックを明示的に呼び出します。

Q10

次のうち『強い依存関係（Strong Dependency）』はどれですか？

  • A他モジュールからサーバーアクションを利用すること。
  • B他モジュールから静的エンティティを利用すること。
  • C外部の REST API を利用すること。
  • D他モジュールからサービスアクションを利用すること。

正解：A

サーバーアクションはコンパイル時にリンクされる強い依存の典型。プロデューサーの変更を反映するにはコンシューマーの再公開が必要です。サービスアクションは弱い依存です。

Q11

クライアントアクションでリストを繰り返し処理したい。どのツールを使いますか？

  • A'If' ノード
  • B'Cycle' ノード
  • C'For Each' ループノード
  • D'ListFilter' アクション

正解：C

'For Each' はリストの各レコードを順に処理する標準のループ要素で、ループ内で 'Current' レコードを提供します（If は分岐、ListFilter は抽出）。

Q12

次のうち『グローバル例外ハンドラー』はどれですか？

  • AC# コード内の Try-Catch ブロック
  • B'OnApplicationReady' イベント
  • Cサーバーアクション内の 'Error' フロー
  • D'Common' UI フロー内で定義された 'OnException' アクション

正解：D

Reactive では Common UI フローの OnException が、未処理のクライアント側例外のキャッチオールハンドラーとして機能します。

Q13

'CustomerDetail' 画面で使う 'Phone' ブロックが電話番号を編集します。保存時に画面の 'LastUpdated' を更新したい。ブロックはどう画面に伝える？

  • Aブロックが直接画面の LastUpdated 変数を更新する。
  • Bブロックが画面の GetCustomer 集計をリフレッシュする。
  • Cブロックが（例：PhoneUpdated）イベントをトリガーし、画面がそれをハンドルする。
  • D画面がブロックの DB テーブルを毎秒チェックする。

正解：C

ブロックは親のスコープに直接アクセスできません。子がイベントを発行→親がキャッチして処理、というイベント駆動が標準で、疎結合を保てます。

Q14

次のうち、集計（Aggregate）の有効な出力はどれですか？

  • Aレコード (Record)
  • B整数 (Integer)
  • C真偽値 (Boolean)
  • Dリスト (List)

正解：D

集計の出力は常にレコードのリスト（List）です（結果が1件や0件でも同じ）。データは AggregateName.List でアクセスします。

Q15

'ExcelToRecordList' で、Excel の列名 'BirthDate' と Structure の属性名 'DateOfBirth' が異なります。どうなりますか？

  • AExcel の最初の列が名前に関係なく最初の属性にマッピングされる。
  • Bその列は無視され、結果のリストでは該当属性が空（デフォルト値）になる。
  • C'Column Name Mismatch' エラーが発生する。
  • DOutSystems が賢く両方日付型としてマッピングする。

正解：B

マッピングはヘッダ名の厳密な文字列一致に基づきます。名前が一致しない列のデータはインポートされず、null/デフォルト値になります（位置や型での暗黙マッチはしません）。

Q16

特定画面へのアクセスを 'Manager' ロールのユーザーだけに制限したい。どう実現しますか？

  • Aマネージャー用の別モジュールを作る。
  • BOnInitialize で 'CheckManagerRole' を使う。
  • C画面内に 'If' ウィジェットを追加して内容を隠す。
  • D画面の 'Roles' プロパティタブで 'Manager' ロールのチェックボックスをチェックする。

正解：D

画面レベルのセキュリティは宣言的。Roles タブでロールをチェックすると、サーバーレベルで不正ユーザーを自動的にリダイレクトします。If で隠すだけではロジックやデータ取得は動くため安全性が低いです。

Q17

List ウィジェットで『無限スクロール』を実現したい。次データの取得はどうトリガーする？

  • A5秒ごとにタイマーでデータ取得する。
  • BList（または Content コンテナ）の 'On Scroll Ending' イベントで MaxRecords を増やしリフレッシュする。
  • COnInitialize で MaxRecords を1,000,000に増やす。
  • D'Pagination' ウィジェットを使う。

正解：B

'On Scroll Ending' でページ末尾到達を検知し、Aggregate の MaxRecords を段階的に増やしてリフレッシュするのが無限スクロールの標準パターンです。

Q18

'ListFilter' システムアクションの結果は？

  • A元のリストを変更し、条件を満たさない要素を削除する。
  • B最初に一致する要素のインデックスを返す。
  • C条件を満たす要素のみを含む新しいリストを返す。
  • D要素が存在するかを示す Boolean を返す。

正解：C

ListFilter は非破壊的で、ソースリストは変更せず、条件に合う要素だけの新しいリストを返します（インデックスは ListIndexOf、存在確認は ListAny）。

Q19

次のうち、『クライアント変数』に保存できないデータ型は？

  • AEntity Identifier
  • BText
  • CBinary Data
  • DInteger

正解：C

クライアント変数はブラウザのローカルストレージに保存されるため、Binary Data のような大きな/複雑な型は非対応。少量の基本型設定データ向けです。

Q20

REST API の 'OnAfterResponse' コールバックで、レスポンス本文をロジックに渡す前に変更したい。どうする？

  • AResponse の ResponseText を操作する（HTTPRequestHandler の応答カスタマイズを含む）。
  • B'JSON Deserialize' ウィジェットを使う。
  • C'SetStatusCode' アクションを使う。
  • DOnAfterResponse 内で本文を変更することは不可能。

正解：A

OnAfterResponse では Response オブジェクトにアクセスでき、ResponseText を変更することでデシリアライズ前に本文をインターセプト・変換（例：復号）できます。SetStatusCode はステータス変更用です。

Q21

Sanitization API の 'SanitizeHtml' メソッドの主な目的は？

  • A< や > のような特殊文字を &lt; や &gt; にエンコードする。
  • BHTML から悪意あるスクリプト（JavaScript）や危険なタグを除去し XSS を防止する。
  • CHTML を適切なインデントで整形する。
  • DHTML をプレーンテキストに変換する。

正解：B

SanitizeHtml は <script> タグや onclick 等の危険な属性を除去しつつ、太字や表など安全な書式は残します。特殊文字エンコードは EncodeHtml の役割です。

Q22

'Date' 型の値を 'DD-MM-YYYY' 形式で表示したい。どの組み込み関数を使う？

  • ADateToString(Date)
  • BFormatDate(Date, "dd-MM-yyyy")
  • CTextToDate(Date)
  • DFormatText(Date, "Date")

正解：B

FormatDate は Date を指定書式（dd-MM-yyyy 等）に変換する標準関数。DateToString はサーバー既定書式、TextToDate は逆変換です。

Q23

ユニットテスト（例：BDDFramework）で条件を正しくアサーションする方法は？

  • A'If' ノードで条件が False の場合に例外を発生させる。
  • Bテスト拡張の 'Assert' アクション（AssertTrue/AssertEquals 等）を使う。
  • C条件が失敗したらメッセージを表示する。
  • D結果をテキストファイルに書き込む。

正解：B

テストフレームワークが提供する Assert 系アクションで期待値と実測値を検証します。失敗するとテストが失敗としてマークされ、レポート UI と連携します。

Q24

タイマーが既に実行中の場合の同時実行防止を、OutSystems はデフォルトでどう扱いますか？

  • A複数回トリガーされても常に並行して実行される。
  • B自動的に同じタイマーの並行実行を防ぎ、実行中に新しいスケジュールが入ると待機させる。
  • C自分で DB の 'IsRunning' フラグを確認する必要がある。
  • D'SingleThread' 拡張を使う必要がある。

正解：B

スケジューラは異なるタイマーは並行実行しますが、同一タイマーはロック（Is_Running_Since/By）で直列化され、実行中なら次の実行は前の完了を待ちます。

Q25

Discovery（アーキテクチャキャンバス）でモジュールを 'Core' から 'Foundation' に移動する操作の意味は？

  • Aモジュールの公開を防ぐ。
  • Bサーバー上のモジュールのフォルダを変更する。
  • C自動的にコードをリファクタして上位参照を除去する。
  • Dアーキテクチャ検証レポート上で論理的に再分類する（物理移動やコード変更はしない）。

正解：D

Discovery はガバナンス/検証ツール。分類を変えると、その層に基づいて違反（例：Foundation がエンドユーザ層を参照）を警告しますが、コードのリファクタや物理移動は行いません。

Q26

ユーザーがブラウザタブを閉じるたびにログアウトされます。ログイン状態を維持するには？

  • A'Login' サーバーアクション内で、User_Login の RememberLogin 入力を True に設定する。
  • Bセッション変数の IsPersistent プロパティを True に設定する。
  • CService Center でセッションのタイムアウトを延長する。
  • Dクライアント変数にパスワードを保存する。

正解：A

User_Login の RememberLogin=True にすると永続クッキー（リメンバーミー）を発行し、ブラウザを閉じても認証状態を保持します（IsPersistent というプロパティは存在しません）。

Q27

Service Studio のロジックで静的エンティティのレコードを削除しようとするとどうなりますか？

  • ASQL クエリで削除する必要がある。
  • Bレコードは削除されるが次の公開後に再表示される。
  • Cレコードは『非アクティブ』とマークされ DB に残る。

正解：D

静的エンティティはメタデータで、レコードのライフサイクルは公開（デプロイ）で管理されます。実行時の Create/Update/Delete アクションは生成されません。

Q28

『ドロップダウン』ウィジェットの有効なソースは次のうちどれですか？

ドロップダウンはリストにバインドするウィジェット。集計が返すレコードのリストをループして選択肢を生成します。

Q29

サーバーアクションで現在ログイン中のユーザーの識別子を取得する関数は？

Q30

アプリでカスタムアイコンを使う最良の方法は？

  • A画像を Base64 文字列に変換し Text ウィジェットに貼る。
  • B外部 URL（例：imgur.com）にリンクする。

ロゴやアイコン等の静的資産は Data タブの Images フォルダに追加するのが標準。アプリと共にデプロイされ、ブラウザ/CDN にキャッシュされます。

Q31

サーバーアクション 'CalculateTax'（Public=Yes）のロジックだけ変更し、入出力シグニチャは変えませんでした。呼び出し側モジュールはどうなりますか？

  • Aリパブリッシュせず動き続ける（Strong Dependency のため、変更反映にはコンシューマーの再公開が必要）。
  • B自動的に最新ロジックがすぐ反映される。
  • Cプラットフォームが自動的に全コンシューマーを再公開する。
  • Dコンシューマーが壊れて実行時エラーをスローする。

正解：A

サーバーアクションは埋め込み（DLL リンク）扱い。シグニチャが同じなら壊れませんが、古いロジックが動き続けるため、新ロジック反映にはコンシューマーの再公開が必要です（サービスアクションなら不要）。

Q32

『Button』ウィジェットの急激な連続クリック（ダブルサブミッション）を防ぐには？

  • A'Ajax Submit' メソッドを使う。
  • B'Single Click' プロパティを True にする。
  • C目的地のタイムアウトを増やす。
  • D標準機能にはないため、Logic（ボタン無効化やロード表示のオーバーレイ）を画面アクション内で実装する。

正解：D

標準のボタンに既定の Disable on Click はありません。クリック時に IsLoading=True にして Enabled を not IsLoading にバインド→処理→IsLoading=False に戻す、が標準パターンです。

Q33

エンティティ属性の 'Is Default' プロパティの目的は？

  • A属性が必須かどうかを決める。
  • B標準のプロパティではない（属性には 'Default Value' プロパティがある）。
  • C属性が主キーかどうかを決める。
  • D属性をエンティティの表示ラベルに設定する。

正解：B

トリック問題。『Is Default』というプロパティは存在せず、属性には初期値を設定する『Default Value』があります（表示ラベルはエンティティの Label Attribute）。

Q34

クライアントアクションがサーバーアクションを呼び、リクエスト送信直後・レスポンス受信前にネットワークが切断されました。何が起こりますか？

  • A接続が復旧するまでアプリがフリーズする。
  • Bクライアントアクションが通信例外を受け取る。
  • Cサーバーアクションが自動的に3回リトライする。
  • Dサーバーはロールバックするがクライアントには通知されない。

正解：B

サーバーに到達できない/接続が切れると、フレームワークはクライアント側で Communication Exception を発生させます。これは OnException で処理でき、オフライン表示等に使えます。

Q35

リスト内で使うブロックから親画面にイベントをトリガーし、アイテムの 'Id' を渡したい。イベントはどう定義しますか？

  • Aイベントはデータを渡せない。親が DB から取得する。
  • Bイベントの定義に 'Id' という入力パラメータを追加する。
  • Cイベントをトリガーし、親がブロックの入力パラメータを直接アクセスする。
  • Dクライアント変数に Id を保存してからイベントをトリガーする。

正解：B

イベントには入力パラメータ（引数）を持たせられ、子から親へデータを渡せます。イベント呼び出し時にその値を引数として渡します。

Q36

'Timer' から呼ばれる 'Server Action' にブレークポイントを設定しましたが、Timer 実行時にヒットしません。理由は？

  • Bブレークポイントは、デバッグ中モジュールのコンテキスト（ユーザーセッション）内で実行されたときのみ機能し、スケジューラーの別セッションでは機能しない。

Q37

'Site Properties' と 'Session Variables' について正しいものは？

  • ASite Properties はブラウザに、Session Variables は DB に格納される。
  • B両方ともすべてのユーザー間で共有される。
  • C両方とも各ユーザーのセッションに固有である。

Q38

'User' エンティティを 'Employee' の詳細で拡張し、1対1リレーションにします。'Employee' の 'Id' 属性のデータ型は？

  • BEmployee Identifier
  • CText
  • DInteger

正解：A

1対1拡張では、拡張側（Employee）の主キーを元エンティティ（User）の識別子型にします。これで Employee.Id が主キー兼 User への外部キーになります。

Q39

'User' と 'Profile' が1対1（各 User に正確に1つの Profile）。OutSystems でどうモデル化しますか？

  • AUser に 'ProfileId' を追加し、Profile に 'UserId' を追加する。
  • B'UserProfile' という第3のエンティティを作り 'UserId' と 'ProfileId' を含める。
  • C'Profile' の 'Id' 属性を 'User Identifier' 型にする。
  • DStatic Entity でマッピングする。

正解：C

1対1では、二次エンティティ（Profile）の主キーを元（User）の識別子型にして主キー値を共有します。相互に FK を持つと循環/重複キーになり、第3エンティティは多対多向けです。

Q40

Producer モジュールのエンティティの 'Expose Read Only' を 'Yes' にした結果は？

Expose Read Only=Yes は、コンシューマーに Select（読み取り）を許可し Insert/Update/Delete を拒否します。書き込みは提供元が用意した公開アクション経由に限定され、整合性を保護できます。
