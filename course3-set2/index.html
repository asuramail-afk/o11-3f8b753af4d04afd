Q1

リアクティブ Web アプリで『保存』ボタンがサーバーアクションを呼びます。実行中に処理中であることを視覚的に伝える、OutSystems のデフォルトの挙動は？

※画面遷移時はトップバーが出ますが、画面内のボタンから呼ぶ（ナビゲーションを伴わない）サーバーアクションに対する汎用の自動スピナーは既定では保証されず、開発者が独自に用意することが多い点に注意してください。

  • Aボタンのテキストが自動的に『Loading...』に変わる。
  • B画面が自動的にロックされグレーアウトされる。
  • C手動でローカル変数 IsLoading を切り替え、スピナーにバインドする必要がある。
  • Dサーバー呼び出し中、画面上部にロードバー（プログレスバー）を自動表示する。

正解：D

Reactive Web アプリでは、サーバーとの通信中に画面上部へ青いプログレスバー（ロードバー）が自動表示されるのがデフォルトの挙動です。画面のロックやボタン文言の変更は自動では行われず、独自のスピナー実装も必須ではありません。

Q2

100通のメールに for each でメール送信中、50通目で例外が発生し、ループ内に例外ハンドラーがある場合どうなりますか？

  • A残りの50通が黙って送信される。
  • Bアプリケーションがクラッシュする。
  • Cループが停止し、トランザクションがロールバックされる。
  • D例外がキャッチされハンドラーのロジック（例：エラーログ）が実行され、その後ループは51通目に進む（継続可能に構成されている場合）。

正解：D

ループ内（またはループから呼ぶラッパーアクション内）に Exception Handler があれば、例外はそこでキャッチされてログ出力などの処理が実行され、ループは次の要素（51通目）へ進めます。1件の失敗で全体を止めない「エラー継続」パターンの基本です。

Q3

フォームで『SubscribeToNewsletter』がオンのときだけ『Email』を必須にしたい。条件付き検証の実装は？

  • AEmail の Mandatory を False にし、保存画面アクション内で条件を確認し、満たさない場合 Input.Valid = False を設定する。
  • Bチェックボックスの On Change で Email の Mandatory を切り替える。
  • CEmail の Mandatory をブール式 SubscribeToNewsletter に設定する。
  • Dシナリオごとに別々の保存ボタンを作る。

正解：C

Reactive の入力ウィジェットの Mandatory プロパティはブール式を直接受け付けるため、Mandatory に SubscribeToNewsletter を設定するのが最も宣言的で簡潔な実装です。保存アクション内で Valid を手動設定する方法（A）も動作しますが、遠回りになります。

Q4

『4 Layer Canvas』アーキテクチャで、汎用の『SendEmail』ラッパーアクションはどこに配置すべき？

  • Aオーケストレーション層
  • Bエンドユーザ層
  • CFoundation 層（ライブラリ）
  • Dコア層

正解：C

メール送信ラッパーのような、特定のビジネスに依存しない技術的ユーティリティは Foundation 層（ライブラリ）に配置します。これにより上位のコア層やエンドユーザ層のどのモジュールからも再利用できます。

Q5

DB に保存された PDF をユーザーがダウンロードできるようにします。リアクティブ Web でのダウンロードロジックの実装は？

  • Aサーバーアクション内で 'Download' ノードを使用する。
  • B画面アクション（クライアント側）のロジックフロー内で 'Download' ノードを使用する。
  • Cリアクティブ Web ではファイルをダウンロードできない。
  • DDB を指す 'External Site' ノードを使用する。

正解：B

Reactive Web では Download ノードはクライアント側、つまり画面アクション（Client Action）のフロー内で使用します。サーバーから取得したバイナリとファイル名を渡すと、ブラウザ標準のダウンロードが開始されます。

Q6

クライアントアクションからトリガーされるサーバーアクションのデフォルトタイムアウトは？

  • A100秒
  • B60秒
  • C無制限
  • D10秒

正解：D

クライアントからのサーバー呼び出しには、モジュールの Server Request Timeout（既定10秒）が適用されます。10秒以内に応答がないと Communication Exception が発生するため、長時間処理は Timer などにオフロードするのが定石です。

Q7

SQL クエリの入力パラメータ 'SearchKeyword' を、SQL インジェクションを防いで使うには？

  • AOutSystems の SQL は構文に関係なく自動的にインジェクションに免疫がある。
  • B送信前に EncodeHtml でサニタイズする。
  • C'@SearchKeyword' 構文（クエリパラメータ）で使用する。
  • D'@SearchKeyword' を使うが Expand Inline を有効にする。

正解：C

@SearchKeyword のようにクエリパラメータとして使えば、DB エンジンは入力をリテラル値として扱い、SQL コードとして解釈しないためインジェクションを防げます。Expand Inline を有効にするとテキストがそのまま SQL に埋め込まれ、脆弱性の原因になります。

Q8

自動生成される『Check<Role>』アクションの目的は？

  • Aユーザーからロールを削除する。
  • Bロールをユーザーに割り当てる。
  • C現在ログイン中のユーザーがその特定のロールを持つか確認する。
  • Dロールが DB に存在するか確認する。

正解：C

自動生成される Check<RoleName>Role 関数は、現在ログイン中のユーザー（指定した UserId）がそのロールを持っていれば True を返します。ロジック内でセキュリティをプログラム的に確認する主要な手段で、付与は Grant、剥奪は Revoke の各アクションが担当します。

Q9

リアクティブ画面のローカル変数 'IsFormValid' が False に変わったとき、すぐに警告を表示したい。実装は？

  • A'Data Action' で変数を監視する。
  • B変数の変化だけではロジックをトリガーできない。変数を変更した場所で明示的に副作用のロジックを呼ぶ必要がある。
  • C'OnRender' で変数値をチェックする。
  • D変数を 'If' ウィジェットにバインドし、その中にメッセージを置く。

正解：B

OutSystems 11 には変数の変更を監視するウォッチャーの仕組みはありません。そのため、変数を変更した箇所で、続けてメッセージ表示などの副作用ロジックを明示的に呼び出す必要があります。

Q10

次のうち『強い依存関係（Strong Dependency）』はどれですか？

  • A他モジュールからサーバーアクションを利用すること。
  • B他モジュールから静的エンティティを利用すること。
  • C外部の REST API を利用すること。
  • D他モジュールからサービスアクションを利用すること。

正解：A

他モジュールの Server Action を参照するのは、コンパイル時にリンクされる Strong Dependency の典型です。プロデューサー側の変更を反映するにはコンシューマーの再公開が必要になります。一方、Service Action や REST API は実行時に解決される弱い依存です。

Q11

クライアントアクションでリストを繰り返し処理したい。どのツールを使いますか？

  • A'If' ノード
  • B'Cycle' ノード
  • C'For Each' ループノード
  • D'ListFilter' アクション

正解：C

リストの各レコードを順番に処理するには For Each ノードを使います。ループ内では Current プロパティで現在のレコードにアクセスできます。If は条件分岐、ListFilter は条件に合う要素の抽出であり、反復処理そのものではありません。

Q12

次のうち『グローバル例外ハンドラー』はどれですか？

  • AC# コード内の Try-Catch ブロック
  • B'OnApplicationReady' イベント
  • Cサーバーアクション内の 'Error' フロー
  • D'Common' UI フロー内で定義された 'OnException' アクション

正解：D

Reactive では、Common UI フローに定義した OnException アクションが、画面側で処理されなかったすべての例外を受け止めるグローバル Exception Handler として機能します。

Q13

'CustomerDetail' 画面で使う 'Phone' ブロックが電話番号を編集します。保存時に画面の 'LastUpdated' を更新したい。ブロックはどう画面に伝える？

  • Aブロックが直接画面の LastUpdated 変数を更新する。
  • Bブロックが画面の GetCustomer Aggregateをリフレッシュする。
  • Cブロックが（例：PhoneUpdated）イベントをトリガーし、画面がそれをハンドルする。
  • D画面がブロックの DB テーブルを毎秒チェックする。

正解：C

Web Block は親画面のスコープに直接アクセスできないため、子ブロックがイベント（例：PhoneUpdated）をトリガーし、親画面がそれをハンドルして LastUpdated を更新するのが標準です。このイベント駆動の連携により疎結合を保てます。

Q14

次のうち、Aggregateの有効な出力はどれですか？

  • Aレコード (Record)
  • B整数 (Integer)
  • C真偽値 (Boolean)
  • Dリスト (List)

正解：D

Aggregate の出力は常にレコードのリスト（List）です。結果が1件や0件でも同じで、データには AggregateName.List でアクセスします。単一の Record や Integer を直接返すことはありません。

Q15

'ExcelToRecordList' で、Excel の列名 'BirthDate' と Structure の属性名 'DateOfBirth' が異なります。どうなりますか？

  • AExcel の最初の列が名前に関係なく最初の属性にマッピングされる。
  • Bその列は無視され、結果のリストでは該当属性が空（デフォルト値）になる。
  • C'Column Name Mismatch' エラーが発生する。
  • DOutSystems が賢く両方日付型としてマッピングする。

正解：B

ExcelToRecordList のマッピングは、ヘッダ名と Structure の属性名の厳密な文字列一致で行われます。名前が一致しない列は無視され、該当属性はデフォルト値（空）のままになります。列の位置や型による暗黙のマッチングは行われません。

Q16

特定画面へのアクセスを 'Manager' ロールのユーザーだけに制限したい。どう実現しますか？

  • Aマネージャー用の別モジュールを作る。
  • BOnInitialize で 'CheckManagerRole' を使う。
  • C画面内に 'If' ウィジェットを追加して内容を隠す。
  • D画面の 'Roles' プロパティタブで 'Manager' ロールのチェックボックスをチェックする。

正解：D

画面レベルのセキュリティは宣言的に設定します。画面の Roles プロパティで Manager ロールにチェックを入れると、権限のないユーザーはサーバー側で自動的にブロック・リダイレクトされます。If ウィジェットで内容を隠すだけではデータ取得やロジックは実行されるため安全ではありません。

Q17

List ウィジェットで『無限スクロール』を実現したい。次データの取得はどうトリガーする？

  • A5秒ごとにタイマーでデータ取得する。
  • BList（または Content コンテナ）の 'On Scroll Ending' イベントで MaxRecords を増やしリフレッシュする。
  • COnInitialize で MaxRecords を1,000,000に増やす。
  • D'Pagination' ウィジェットを使う。

正解：B

無限スクロールの標準パターンは、On Scroll Ending イベントでリスト末尾への到達を検知し、Aggregate の Max Records を段階的に増やしてリフレッシュする方法です。最初から大量件数を取得したり Timer でポーリングしたりするのは非効率です。

Q18

'ListFilter' システムアクションの結果は？

  • A元のリストを変更し、条件を満たさない要素を削除する。
  • B最初に一致する要素のインデックスを返す。
  • C条件を満たす要素のみを含む新しいリストを返す。
  • D要素が存在するかを示す Boolean を返す。

正解：C

ListFilter は非破壊的なアクションで、元のリストは変更せず、条件を満たす要素だけを含む新しいリストを返します。インデックス取得は ListIndexOf、存在確認は ListAny の役割です。

Q19

次のうち、『クライアント変数』に保存できないデータ型は？

  • AEntity Identifier
  • BText
  • CBinary Data
  • DInteger

正解：C

Client Variable はブラウザのローカルストレージに保存されるため、Text・Integer・Entity Identifier などの基本型のみ対応で、Binary Data は保存できません。少量の設定データ向けの仕組みです。

Q20

REST API の 'OnAfterResponse' コールバックで、レスポンス本文をロジックに渡す前に変更したい。どうする？

  • AResponse の ResponseText を操作する（HTTPRequestHandler の応答カスタマイズを含む）。
  • B'JSON Deserialize' ウィジェットを使う。
  • C'SetStatusCode' アクションを使う。
  • DOnAfterResponse 内で本文を変更することは不可能。

正解：A

OnAfterResponse では Response オブジェクトにアクセスでき、ResponseText を書き換えることでデシリアライズ前に本文をインターセプト・変換（例：復号や整形）できます。SetStatusCode はステータスコード変更用で、本文の変更には使いません。

Q21

Sanitization API の 'SanitizeHtml' メソッドの主な目的は？

  • A< や > のような特殊文字を &lt; や &gt; にエンコードする。
  • BHTML から悪意あるスクリプト（JavaScript）や危険なタグを除去し XSS を防止する。
  • CHTML を適切なインデントで整形する。
  • DHTML をプレーンテキストに変換する。

正解：B

SanitizeHtml は、HTML から script タグや onclick などの危険な属性を除去して XSS を防止しつつ、太字や表といった安全な書式は残します。特殊文字を実体参照に変換するのは EncodeHtml の役割です。

Q22

'Date' 型の値を 'DD-MM-YYYY' 形式で表示したい。どの組み込み関数を使う？

  • ADateToString(Date)
  • BFormatDateTime(Date, "dd-MM-yyyy")
  • CTextToDate(Date)
  • DFormatText(Date, "Date")

正解：B

FormatDateTime(Date, "dd-MM-yyyy") のように書式文字列を指定して変換するのが正解です。DateToString はサーバーの既定書式でしか出力できず、TextToDate はテキストから日付への逆変換です。

Q23

ユニットテスト（例：BDDFramework）で条件を正しくアサーションする方法は？

  • A'If' ノードで条件が False の場合に例外を発生させる。
  • Bテスト拡張の 'Assert' アクション（AssertTrue/AssertEquals 等）を使う。
  • C条件が失敗したらメッセージを表示する。
  • D結果をテキストファイルに書き込む。

正解：B

BDDFramework などのテストフレームワークが提供する Assert 系アクション（AssertTrue、AssertEquals など）で期待値と実測値を検証します。アサーションが失敗するとテストが失敗としてマークされ、レポート UI に反映されます。

Q24

タイマーが既に実行中の場合の同時実行防止を、OutSystems はデフォルトでどう扱いますか？

  • A複数回トリガーされても常に並行して実行される。
  • B自動的に同じタイマーの並行実行を防ぎ、実行中に新しいスケジュールが入ると待機させる。
  • C自分で DB の 'IsRunning' フラグを確認する必要がある。
  • D'SingleThread' 拡張を使う必要がある。

正解：B

スケジューラは異なる Timer は並行実行しますが、同一 Timer はプラットフォームが自動的にロックして直列化します。実行中に次のスケジュールが来た場合は、前の実行の完了を待ってから開始されるため、自前の排他制御は不要です。

Q25

Discovery（アーキテクチャキャンバス）でモジュールを 'Core' から 'Foundation' に移動する操作の意味は？

  • Aモジュールの公開を防ぐ。
  • Bサーバー上のモジュールのフォルダを変更する。
  • C自動的にコードをリファクタして上位参照を除去する。
  • Dアーキテクチャ検証レポート上で論理的に再分類する（物理移動やコード変更はしない）。

正解：D

Discovery はガバナンス・検証ツールであり、キャンバス上の移動はアーキテクチャレポート上の論理的な再分類にすぎません。分類に基づいてルール違反（例：Foundation が上位層を参照）を警告しますが、コードのリファクタリングや物理的な移動は一切行いません。

Q26

ユーザーがブラウザタブを閉じるたびにログアウトされます。ログイン状態を維持するには？

  • A'Login' サーバーアクション内で、User_Login の RememberLogin 入力を True に設定する。
  • Bセッション変数の IsPersistent プロパティを True に設定する。
  • CService Center でセッションのタイムアウトを延長する。
  • Dクライアント変数にパスワードを保存する。

正解：A

User_Login アクションの RememberLogin 入力を True にすると永続クッキー（リメンバーミー）が発行され、ブラウザを閉じても認証状態が維持されます。Session Variable に IsPersistent というプロパティは存在せず、パスワードのクライアント側保存は重大なセキュリティリスクです。

Q27

Service Studio のロジックで静的エンティティのレコードを削除しようとするとどうなりますか？

  • ASQL クエリで削除する必要がある。
  • Bレコードは削除されるが次の公開後に再表示される。
  • Cレコードは『非アクティブ』とマークされ DB に残る。
  • D静的エンティティのレコードはプログラムで削除できない（設計時にのみ管理）。

正解：D

Static Entity のレコードは設計時のメタデータとして扱われ、公開（デプロイ）によってのみ管理されます。そのため Create/Update/Delete といった Entity Action は生成されず、実行時にプログラムから削除することはできません。

Q28

『ドロップダウン』ウィジェットの有効なソースは次のうちどれですか？

  • A静的エンティティ識別子。
  • BStructureの定義。
  • Cレコードのリストを返すAggregate。

正解：C

ドロップダウンはリストをソースに取るウィジェットで、Aggregate が返すレコードのリストを反復して選択肢を生成します。単一の識別子や文字列変数、Structure の定義そのものはソースにできません。

Q29

サーバーアクションで現在ログイン中のユーザーの識別子を取得する関数は？

  • AGetUserId()
  • BGetCurrentUser()
  • CSession.UserId
  • DUser.Id

正解：A

GetUserId() は現在ログイン中のユーザーの識別子を返す組み込み関数で、クライアント側・サーバー側どちらのロジックでも使用できます。他の選択肢の関数やプロパティは存在しません。

Q30

アプリでカスタムアイコンを使う最良の方法は？

  • A画像を Base64 文字列に変換し Text ウィジェットに貼る。
  • B外部 URL（例：imgur.com）にリンクする。
  • C画像を Data タブの 'Images' フォルダにインポートし、Image ウィジェット（または CSS）で使う。

正解：C

ロゴやアイコンなどの静的アセットは、Data タブの Images フォルダにインポートして Image ウィジェットや CSS から参照するのが標準です。アプリと一緒にデプロイされ、ブラウザや CDN でキャッシュされるためパフォーマンスにも優れます。

Q31

サーバーアクション 'CalculateTax'（Public=Yes）のロジックだけ変更し、入出力シグニチャは変えませんでした。呼び出し側モジュールはどうなりますか？

  • Aリパブリッシュせず動き続ける（Strong Dependency のため、変更反映にはコンシューマーの再公開が必要）。
  • B自動的に最新ロジックがすぐ反映される。
  • Cプラットフォームが自動的に全コンシューマーを再公開する。
  • Dコンシューマーが壊れて実行時エラーをスローする。

正解：A

Server Action は Strong Dependency としてコンシューマーにコンパイル時リンクされるため、シグニチャが同じなら壊れずに動き続けますが、実行されるのは古いロジックのままです。新しいロジックを反映するにはコンシューマーの再公開が必要です（Service Action なら再公開不要）。

Q32

『Button』ウィジェットの急激な連続クリック（ダブルサブミッション）を防ぐには？

  • A'Ajax Submit' メソッドを使う。
  • B'Single Click' プロパティを True にする。
  • C目的地のタイムアウトを増やす。
  • D標準機能にはないため、Logic（ボタン無効化やロード表示のオーバーレイ）を画面アクション内で実装する。

正解：D

Q34

Q35

リスト内で使うブロックから親画面にイベントをトリガーし、アイテムの 'Id' を渡したい。イベントはどう定義しますか？

Q36

'Timer' から呼ばれる 'Server Action' にブレークポイントを設定しましたが、Timer 実行時にヒットしません。理由は？

Q37

Q38

Q39

'User' と 'Profile' が1対1（各 User に正確に1つの Profile）。OutSystems でどうモデル化しますか？

Q40

Producer モジュールのエンティティの 'Expose Read Only' を 'Yes' にした結果は？

Q48
