Q1

標準の OutSystems アプリ（Users モジュール）で 'Group' を定義するとき、'Group' は主に何に使われますか？

  • AService Studio 内でモジュールをグループ化するため。
  • B複数のユーザーに一度に Roles を割り当てるため。
  • Cチャットグループを作成するため。
  • Dサーバーアクションをグループ化するため。

正解：B

Group は複数のユーザーをまとめて Role を割り当てるためのコンテナです。例えば「IT グループ」に管理者のロールを割り当てると、そのグループに属する全ユーザーがそのロールを持つことになり、権限管理を効率化できます。

Q2

エンティティ 'Ticket' の 'StatusId' について、Table で ID ではなくラベル（例："Open"）を効率的に表示するには？

  • A行の式で GetStatus(Ticket.StatusId).Label を使う。
  • Bクライアント変数にステータス一覧を保存し JavaScript でルックアップする。
  • Cラベルは表示できず ID のみ表示できる。
  • DTicket 取得の Aggregate に 'Status'（Join Ticket.StatusId = Status.Id）を追加し、Table で 'Status.Label' を使う。

正解：D

Aggregate に Status Entity を Join として追加すれば、単一のクエリでラベルを取得でき最も効率的です。行ごとの式で Server Action を呼び出すと、行数分のクエリが発生する N+1 問題となり、性能が大きく劣化します。

Q3

ブロックにプレースホルダーがあり、利用者がそこに何もウィジェットを置かなかった場合どうなりますか？

  • A画面を公開できなくなる。
  • Bブロックがエラーをスローする。
  • C何も表示されずスペースが折りたたまれる（プレースホルダーにデフォルトコンテンツが定義されている場合を除く）。
  • Dそのスペースに赤い『X』が表示される。

正解：C

プレースホルダーは任意で埋めるスロットなので、空のままでもエラーにはならず、何も表示されずスペースが折りたたまれます。ただし Web Block 側でプレースホルダー内にウィジェットを置いておくと、それがデフォルトコンテンツとして表示され、利用側で上書きできます。

Q4

'Service Center' → 'Monitoring' → 'Environment Health' ページの主な目的は？

  • AOutSystems のサービス（Deployment Controller、Scheduler など）とデータベースの接続状況を確認する。
  • Bアプリのソースコードを閲覧する。
  • C新しいユーザーを作成する。
  • Dクライアントの PC の CPU 使用率を監視する。

正解：A

Environment Health は、Deployment Controller や Scheduler などの OutSystems サービスとデータベース接続の稼働状況を信号色（緑/黄/赤）で表示するページです。例えば「Timer が動かない」ときにまず確認する場所で、Scheduler サービスが赤なら原因を特定できます。

Q5

Container ウィジェットをブロックで使う有効なシナリオはどれですか？

  • A複数のウィジェットをグループ化して CSS クラスを適用したり 'If' で表示制御したりする。
  • B親画面からウィジェットを渡せるようにする。
  • C画面にコメントを追加する。
  • Dブロックに入力変数を作成する。

正解：A

Container は div としてレンダリングされる構造用ウィジェットで、複数のウィジェットをまとめて CSS クラスを適用したり、If で表示・非表示を切り替えたりするのに使います。親画面からコンテンツを注入するための「穴」を作るのは Container ではなく Placeholder の役目です。

Q6

'Employee' と 'City' を結合し、City が割り当てられていない Employee も含めて全て取得したい。どの結合タイプ？

  • AWith or Without（左外部結合）
  • BOnly With（内部結合）
  • CCross Join（クロス結合）
  • DWith（完全外部結合）

正解：A

With or Without は SQL の Left Join に相当し、左側（Employee）の全レコードを返し、City があれば結合し、なければ null になります。Only With（Inner Join）では両方が一致するレコードのみが返るため、City が未設定の Employee は除外されてしまいます。

Q7

アーキテクチャキャンバスで『循環依存関係』の違反を構成するのは何ですか？

  • Aモジュール A と B が両方ともモジュール C から要素を消費している。
  • Bモジュール A が自分自身を参照している。
  • Cモジュール A が B の公開要素を消費し、B が A の公開要素を消費している（直接または間接的に）。
  • Dモジュール A が B を消費し、B が C を消費している。

正解：C

循環依存は、モジュール A が B の公開要素を消費し、同時に B も（直接または間接的に）A の公開要素を消費している状態です。互いに参照し合うため密結合になり、独立したデプロイができず、参照が壊れる原因になります。A→B→C のような一方向の参照は正常な階層構造です。

Q8

エンティティの 'Text' 属性の最大長は、特に指定しない場合デフォルトで何文字ですか？

  • A100文字
  • B無制限（最大）
  • C50文字
  • D255文字

正解：C

Entity に新しく Text 属性を作成すると、Length の既定値は 50 文字です。必要に応じて手動で拡張しますが、増やし忘れると保存時に 50 文字で切り捨てられるため、典型的なバグの原因になります。

Q9

Reactive Web App のシステムイベント 'On Application Ready' の目的は？

  • A画面がロードされるたびに実行される。
  • Bサーバーが起動したときに実行される。
  • Cユーザーがログインしたときに実行される。
  • Dブラウザにアプリがロードされたときに一度だけ実行され、グローバルな設定処理（例：アナリティクス設定、初期トークン確認）が可能。

正解：D

On Application Ready はアプリがブラウザにロードされたとき、最初の画面がレンダリングされる前に一度だけ実行されるイベントです。アナリティクスの設定やサードパーティスクリプトの初期化、グローバルな既定値の設定など、アプリ全体の初期処理に使います。画面遷移のたびには実行されません。

Q10

『Static Entities（静的エンティティ）』について誤っている記述はどれですか？

  • AStatic Entities はデータベースに格納されている。
  • B'Create' アクションを使って実行時に新しいレコードを追加できる。
  • CStatic Entities はリテラル識別子（例：Status.New）を持つ。
  • DStatic Entities は外部キーとして使用できる。

正解：B

Static Entity のレコードは実行時に Create などで追加・変更できないため、B が誤りです。レコードは Service Studio 上で管理され、モジュールの公開時に反映されます。データベースに格納され、リテラル識別子（例：Status.New）を持ち、外部キーとしても使える、列挙型のような定数データです。

Q11

'JSON Deserialize' ツールの 'Data Type' プロパティに必要な必須要件は？

  • A'Text' または 'Integer' タイプのみを受け付ける。
  • BJSON 文字列から構造を自動的に検出する。
  • CXML スキーマが必要。
  • DJSON 構造に一致する Structure、Entity、または List 定義を提供する必要がある。

正解：D

OutSystems は静的型付けのため、JSON Deserialize では受信する JSON の構造に一致する Structure、Entity、または List の型を Data Type に設計時に指定する必要があります。この型定義が、JSON のプロパティを変数へマッピングするための設計図になります。自動検出はされません。

Q12

ウィジェットの 'Style Classes'（スタイルクラス）プロパティからテーマの CSS クラスを参照する正しい構文は？

  • A".my-class"（ドット付き）
  • Bクラス名をそのまま入力（例：my-class）
  • CClass("my-class")
  • D"#my-class"

正解：B

Style Classes プロパティは HTML の class 属性に直接対応するため、クラス名だけをそのまま入力します（ドットや # は不要、複数指定はスペース区切り）。ドット（.）は CSS 側でクラスを定義するときの記法、# は ID セレクタ用です。

Q13

サーバーとは異なるタイムゾーンで日時を表示したいとき、最も役立つのは？

  • A単純に5時間引き算する。
  • Bサーバーのタイムゾーン設定を変更する。
  • C'Time Zone' Forge コンポーネント（または利用可能なら ConvertFromTimeZone 関数）。
  • D'FormatDate' 関数を使う。

正解：C

タイムゾーン変換はサマータイム（DST）を含めて複雑なため、Forge の Time Zone コンポーネントなどを使い、IANA タイムゾーンに基づいて変換するのが正解です。固定のオフセットを引き算するハードコードは DST に対応できず壊れやすい実装になります。

Q14

ブロックに Boolean の入力パラメータ 'IsActive' があります。これに応じてブロック内コンテナのスタイルを変える最適な方法は？

  • A状態ごとに 'If' で囲んだ2つの異なるコンテナを使う。
  • Bコンテナの 'Style Classes' に式 If(IsActive, "status-active", "status-inactive") を使う。
  • CJavaScript で色を変更する。
  • D'Style' 属性にハードコードした色を直接設定する。

正解：B

Style Classes プロパティは式を受け付けるので、If(IsActive, "status-active", "status-inactive") のように条件で CSS クラスを切り替えるのが最適です。HTML がクリーンに保たれ、スタイルは CSS に集約できます。If で同じコンテナを2つ用意すると DOM が重複し、インラインスタイルの直接指定は保守性を損ないます。

Q15

Service Center の 'Factory' > 'Modules' リストにある 'Publish' ボタンは何をしますか？

  • Aモジュールを削除する。
  • B最後にアップロードしたバージョンのモジュールを再展開する。
  • C以前のバージョンにロールバックする。
  • Dコンピュータから新しいバージョンをアップロードする。

正解：B

Service Center の Publish ボタンは、最後にアップロード済みのバージョンをそのまま再デプロイします。Service Studio を開かずに、参照の問題を解消したり設定変更を反映させたりするのに使います。新しいバージョンのアップロードやロールバックを行うボタンではありません。

Q16

クライアントアクション内でサーバーアクションを呼び出した場合、デフォルトの実行の流れは？

  • Aサーバーアクションは後で実行されるようキューに追加される。
  • Bブラウザに依存する。
  • Cクライアントアクションはサーバーアクションの完了まで待機し（開発者視点で同期的に振る舞い）、次のノードに進む。
  • Dクライアントアクションはすぐ続行し、サーバーアクションはバックグラウンドで実行される。

正解：C

Client Action から Server Action を呼び出すと、フローはその呼び出しノードでレスポンスを待ってから次のノードへ進みます（内部的には非同期通信ですが、開発者視点では同期的に振る舞います）。そのため Server Action の出力を直後のノードでそのまま利用できます。

Q17

モジュール公開時に 'Upgrade Violation' エラーが 'Strong Dependency' メッセージとともに表示されました。何を意味しますか？

  • Aプロデューサーモジュールが削除された。
  • Bサーバーアクションを多く使いすぎている。
  • C循環依存がある。
  • Dプロデューサーの公開アクションの入出力シグニチャを変更したが、コンシューマーをまだリフレッシュ/再公開していない。

正解：D

プロデューサー側で公開 Server Action の入出力シグネチャを変更したのに、コンシューマー側の References をまだリフレッシュ・再公開していない状態を意味します。コンシューマーを開いて依存関係をリフレッシュし、再公開すれば解消します。なお、要素の削除は Missing Reference という別のエラーになります。

Q18

'ListSort' システムアクションはどのように動作しますか？

  • Aデータベースのテーブルの順番を変更する。
  • B指定した属性と方向に基づいて、リストをメモリ内で並べ替える。
  • C新しいリストを作成する。
  • Dリストをフィルタリングする。

正解：B

ListSort は、指定した属性と昇順・降順に基づいてメモリ内のリスト変数を並べ替えるシステムアクションです。データベースのテーブル自体には順序という概念がなく影響しません。リストの絞り込みを行うのは ListFilter です。

Q19

Excel からのブートストラップ機能で、Excel に重複行が含まれていた場合どうなりますか？

  • A自動的に重複をマージする。
  • Bすぐにインポートを停止する。
  • Cエンティティに定義されたインデックス次第。ユニークインデックスがあれば重複は失敗し、なければ挿入される。
  • D既存レコードを更新する。

正解：C

Excel ブートストラップは基本的に Create の繰り返しであり、結果はデータベース制約に従います。Entity にユニークインデックスが定義されていれば重複行の挿入は失敗し、なければそのまま挿入されます。自動的に重複を検出・マージする仕組みはありません。

Q20

リアクティブ Web アプリの検索で、ユーザーが入力を止めてから500ms後に検索したい（Debounce）。どう実現しますか？

  • AInput の OnChange で setTimeout を使う JavaScript ノードを用い、既存のタイムアウトをクリアする。
  • BInput ウィジェットの 'Debounce' プロパティを使う。
  • C500ms ごとに動く Timer を使う。
  • DOnChange 内で 'Sleep' アクションを使う。

正解：A

Debounce は、Input の OnChange から呼ぶ JavaScript ノードで setTimeout を使い、入力のたびに既存のタイムアウトを clearTimeout でクリアする実装が標準です。標準の Input ウィジェットに Debounce プロパティはなく、Timer はサーバー側のバックグラウンド処理用、Sleep は UI をフリーズさせるため不適切です。

Q21

ユーザーがログアウトしたとき、'セッション変数' のデータはどうなりますか？

  • Aセッションが無効になり、次のセッションのためにセッション変数はデフォルト値にリセットされる。
  • Bデータはブラウザを閉じるまで保持される。
  • Cデータは自動的にデータベースに保存される。
  • Dデータは次のユーザーのためにコンピュータ上に残る。

正解：A

Session Variable はユーザーのアクティブなセッションに紐づく一時的な状態です。ログアウトやタイムアウトでセッションが無効化されると破棄され、次のセッションではデフォルト値にリセットされます。データベースに保存されたり、他のユーザーに引き継がれたりすることはありません。

Q22

Switch ウィジェットに Status="New"、Status="Closed"、Otherwise の3ブランチがあります。Status が "New" のとき、どのブランチが実行されますか？

  • AOtherwise がデフォルトなので Otherwise ブランチが実行される。
  • BNew ブランチが実行され、その後 Otherwise にフォールスルーする。
  • C条件が最初に True になるブランチだけが実行される（この場合 Status="New"）。
  • DTrue と評価される全ブランチが実行される。

正解：C

Switch は上から順に条件を評価し、最初に True になったブランチだけを実行して評価を終了します。If-ElseIf-Else のような排他的な分岐であり、プログラミング言語の switch 文のようなフォールスルーはしません。Otherwise はどの条件も True にならなかった場合のみ実行されます。

Q23

GrantRole と CheckRole の主な違いは何ですか？

  • AGrantRole は権限をデータベースに保存し、CheckRole は現在のリクエストで権限を確認する。
  • BGrantRole は一時的で、セッション期間だけ有効。
  • CGrantRole は画面向け、CheckRole はロジック向け。
  • D違いはない。

正解：A

GrantRole はロールの割り当てをデータベースに永続的に書き込む管理操作で、CheckRole は現在のユーザーがそのロールを持つかを検証する読み取り操作です。Grant で付与し、Check をセキュリティガードとしてロジック内で都度確認する、という役割分担になります。

Q24

特定のロール（例：'Approver'）をプログラム的にユーザーに付与するには、どのアクションを使いますか？

  • Aユーザーエンティティを直接更新する。
  • B'CreateUserRole' エンティティアクション。
  • C'AddRoleToUser' システムアクション。
  • D'GrantApproverRole' サーバーアクション。

正解：D

モジュールで 'Approver' というロールを定義すると、プラットフォームが GrantApproverRole / RevokeApproverRole / CheckApproverRole という Server Action を自動生成します。付与には Grant<ロール名>Role(UserId) を使うのが型安全で標準的な方法です。ロールの割り当ては専用のリンクテーブルに保存されるため、User Entity を直接更新する方法は誤りです。
