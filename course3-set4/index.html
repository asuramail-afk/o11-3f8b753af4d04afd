Q1

Service Studio でエンティティを削除して公開すると、データベースのデータはどうなりますか？

  • Aデータは CSV ファイルにアーカイブされる。
  • Bテーブルは DB に残るが、OutSystems はそれをモジュールから切り離す（ソフト削除／デタッチ）。
  • Cテーブルとその全データが即座にドロップ（ハード削除）される。
  • D手動でテーブルを削除しない限り、公開は失敗する。

正解：B

IDE でエンティティを削除してもメタデータ定義が消えるだけで、公開時に DROP TABLE は実行されません（誤削除からの保護）。物理的にテーブルを削除するには DbCleaner API（Entity_DropTable）を使います。

Q2

'ExcelToRecordList' で、Excel の列名が 'Phone Number'（空白あり）、構造属性は 'PhoneNumber'（空白なし）です。自動的にマッピングされますか？

  • Aはい、'Automap' ボタンを使えば自動的にマッピングされる。
  • Bはい、両方が3番目の列ならマッピングされる。
  • Cはい、OutSystems は空白を無視してマッピングする。
  • Dいいえ。名前は正確に一致する必要がある（通常は大文字小文字は無視だが空白は重要）。属性名かヘッダー名を一致させる。

正解：D

マッピングは文字列（名前）ベースで、位置ではありません。'Phone Number' は 'PhoneNumber' と一致せず、その列は無視されます。空白やタイプミスに注意が必要です。

Q3

ダッシュボード画面で、ユーザーがページを更新しなくても5分ごとに自動でデータをリフレッシュしたい。どう実装しますか？

  • AContainer を画面にドラッグし、OnReady クライアントアクションで JavaScript の setInterval を使ってクライアントアクションをトリガーし、集計をリフレッシュする。
  • BLogic タブの 'Timer' プロセスを使う。
  • CAggregate の 'Auto Refresh' プロパティを使う。
  • D'OnRender' イベントでリフレッシュをループさせる。

正解：A

Reactive の自動リフレッシュは JavaScript の setInterval が標準。interval 発火時にクライアントアクションを呼び RefreshAggregate します。メモリリーク防止のため OnDestroy で clearInterval します（Timer はサーバー背景処理、OnRender は無限ループになります）。

Q4

ブロック 'UserInfo' が入力パラメータ 'UserId' を受け取り、内部の Aggregate 'GetProfile' を UserId でフィルタしています。親画面で UserId が変わると、この Aggregate はどうなりますか？

  • A自動的にはリフレッシュされない。'OnParametersChanged' で明示的にリフレッシュする必要がある。
  • Bブロックが破棄されて再作成される。
  • Cアプリケーションがエラーをスローする。
  • DAggregate は自動的にリフレッシュされる。

正解：A

Reactive では入力パラメータが変わっても、それを使う Aggregate は自動再実行されません。OnParametersChanged ライフサイクルイベントで RefreshAggregate を呼ぶ必要があります（従来 Web との重要な違い）。

Q5

Logic Flow でコードにコメントを付ける有効な方法はどれですか？

  • Aノードの名前を // で始める。
  • Bフローエディタにはコメントを追加できない。
  • C背景を右クリックして 'ノートを追加' を選ぶ。
  • Dツールボックスから 'Comment' ツールをフローにドラッグする。

正解：D

Comment ノードをキャンバスに配置すると、テキストの注釈を直接置けて、複雑なロジックの説明やノードの視覚的グループ化に役立ちます。

Q6

画面が非常に遅く、Preparation/OnInitialize の Aggregate が過剰にデータを取得していました。一般的な最適化は？

  • Aサーバーのタイムアウトを増やす。
  • Bページネーション（Max Records）を実装し、取得する属性を最小限にする（使う列だけ取得）。
  • C静的エンティティを使う。
  • Dロジックをクライアント側に移動する。

正解：B

最適化の基本は『取得データを減らす』こと。10件しか見ないのに10,000件取らない、リストに Name だけ表示なら BinaryPhoto を取らない。Aggregate は属性を自動最適化しますが、MaxRecords（行数制限）は開発者が管理します。

Q7

『パーソナル環境（Personal Environment）』とは何ですか？

  • Aアプリケーションのテーマ。
  • B企業環境内のサンドボックス。
  • C学習やプロトタイピングのために個人開発者に提供される無料のクラウドベースの OutSystems 環境。
  • Dラップトップにインストールされたローカルの OutSystems。

正解：C

パーソナル環境は OutSystems がホスティングするフルスタック環境（プラットフォームサーバー＋DB）で、サーバーをインストールせずに学習・試作できます（容量やスリープ制限あり）。

Q8

『Anonymous』ユーザーの『役割（Role）』は何ですか？

  • Aログインしていない任意のユーザーを表す。
  • Bログインしているが名前のないユーザー。
  • C管理者（Administrator）役割。
  • Dロボット専用の役割。

正解：A

Anonymous 役割を画面のセキュリティ設定でチェックすると、認証なしでアクセスできるパブリック画面（ランディング/ログイン画面）を作れます。チェックしないとログイン画面へリダイレクトされます。

Q9

ユーザーが Table ウィジェットの行をクリックしたときにトリガーされるイベントは？

  • A行の設定次第。通常は行や特定のセル／リンクに 'On Click' イベントハンドラーを追加する。
  • BOnRowClick
  • COnSelect
  • DOnChange

正解：A

Table に既定の『行クリック』イベントはありません。行（またはその内部コンテナ）の On Click イベントを明示的にクライアントアクションにバインドする必要があります。

Q10

ライブラリモジュールのサーバーアクションの 'Public' プロパティを 'Yes' にする意味は？

  • Aこのアクションは、環境内の他のモジュールから参照して使用できる。
  • Bこのアクションは、同じアプリ内のモジュールからのみ使用できる。
  • Cこのアクションは、外部ユーザーから匿名で呼び出せる。
  • Dこのアクションは自動的に REST API になる。

正解：A

Public=Yes は要素をファクトリの他の部分に公開し、他モジュールが依存関係を追加できます（アーキテクチャルールの制約下で任意のアプリから）。REST 公開は別途 REST 統合の設定が必要です。

Q11

Service Studio の 'TrueChange' エンジンの役割は？

  • Aコードをリアルタイムで分析し、公開前にエラー・警告・パフォーマンスの問題を検出する。
  • Bアプリケーションのバージョン管理を行う。
  • Cバグを自動的に修正する。
  • DSQL クエリを最適化する。

正解：A

TrueChange はバックグラウンドのコンパイラー／バリデーターで、エラータブに情報を表示し、壊れたコード（型の不一致等）を公開させない即時フィードバックループです（バグの自動修正や SQL 書き換えはしません）。

Q12

'Users' アプリで新しいユーザーを作成するとき、必須のユニーク識別子のフィールドは？

  • AEmail（メールアドレス）
  • BPassword（パスワード）
  • CUsername（ユーザー名）
  • DFull Name（フルネーム）

正解：C

標準の User エンティティは、ログインに使うユニークな Username を必須とします（Username 属性にユニーク制約）。Email もしばしばユニークですが、厳格な制約は Username にあります。

Q13

'Order' エンティティの 'CreatedBy' 属性から、作成したユーザーの名前を Aggregate で取得するには？

  • A式内で 'GetUserName' 関数を使う。
  • BSystem エンティティとは結合できない。
  • CSQL クエリを作成する。
  • D'User' エンティティを Aggregate のソースに追加する。プラットフォームが結合条件（Order.CreatedBy = User.Id）を自動推測する。

正解：D

User システムエンティティを Aggregate のソースにドロップすると、FK 関係（CreatedBy は User 識別子型）を検出して正しい Join を自動作成します（単一クエリで効率的。式で毎行取得すると N+1 問題）。

Q14

エンティティ属性の『必須』(Is Mandatory) プロパティは何に使われますか？

  • A画面上の UI バリデーションだけに影響する。
  • B属性をプライマリキーにする。
  • Cデータを暗号化する。
  • Dデータベース制約（NOT NULL）と OutSystems のバリデーションを強制し、この値なしではレコードを保存できなくする。

正解：D

Is Mandatory=Yes は DB スキーマに NOT NULL 制約を送り、加えて画面の入力を自動的に必須にします。UI だけでなく DB レベルでデータ整合性を守ります。

環境設定（Service Center）の『表示名（Display Name）』プロパティの目的は？

  • C管理者のユーザー名。

Q16

入力フィールドに有効なメールアドレスだけを受け入れたい。最も速く組み込みの方法は？

  • AOnChange イベントで正規表現を使う。
  • B組み込みのメールバリデーションは存在しない。
  • CInput ウィジェットの 'Input Type' プロパティを 'Email' に設定する。
  • D独自のサーバーアクションを作って構文を検証する。

正解：C

Input Type=Email にすると、HTML5 のネイティブなクライアント側検証（赤枠やツールチップ）が追加ロジックなしで働き、モバイルでは適切なキーボードも表示されます。

Q17

公開時に『モジュール A は署名 Y のアクション X を期待するが、モジュール B は署名 Z を提供』というリファレンス/互換性エラーが出ました。どう修正しますか？

  • Aモジュール A を削除して再作成する。
  • BService Center のサービスを再起動する。
  • Cモジュール B（プロデューサ）を開いて変更を元に戻す。
  • Dモジュール A（コンシューマ）を開き『依存関係の管理』で B への参照をリフレッシュしてから A を公開する。

正解：D

コンシューマ（A）はプロデューサ（B）の古い定義を持っています。依存関係をリフレッシュすると契約（定義）が更新され、A が B の変更を認識できます。プロデューサの変更を戻すのは進行の巻き戻しです。

Q18

モバイル対応で、内容を縦に並べ、小さな画面で自動的に積み重なる（スタックする）配置に最適なウィジェットは？

  • A'Columns' ウィジェット（Columns2、Columns3 など）または 'Adaptive Columns'。
  • B'Table' ウィジェット。
  • C固定幅の 'Container' ウィジェット。
  • D'If' ウィジェット。

正解：A

OutSystems UI の Columns はレスポンシブで、デスクトップでは横並び、モバイル/タブレットでは自動的に縦に折りたたまれます。Table は横スクロールが必要になり、固定幅 Container はレスポンシブ性を壊します。

Q19

'Feedback Message' クライアントアクションの目的は？

  • A一時的な通知（トースト）をユーザーに表示する（例：『成功』『エラー』）。

  • C管理者にメールを送信する。

Feedback Message は、操作結果（例：『保存しました』）を操作を妨げず非侵襲的に伝え、自動的に消える一時的なステータス表示の標準手段です（ログ書込は LogMessage）。

Q20

高精度の小数を含む複雑な数学計算を行います。どの変数データ型を使うべきですか？

  • Aテキスト（Text）
  • Bブーリアン（Boolean）
  • C整数（Integer）
  • D小数（Decimal）

正解：D

Decimal は高精度の算術（金額や科学計算）向けに設計され、浮動小数点でよくある誤差を避けます（Integer は小数不可。Currency は基本的に Money 用にフォーマットした Decimal）。

Q21

ロジックフローでレコードを更新中、DB 制約違反で更新が失敗し DB 例外がスローされます。このシナリオをどう処理しますか？

  • A'If' ノードで DB がオンラインかどうかを確認する。
  • Bフローに例外ハンドラーを追加し、例外タイプを 'DatabaseException' に設定してフォールバックのロジック（例：メッセージ表示）を実装する。
  • CC# コードの 'Try-Catch' ブロックで更新を囲む。
  • Dデータベースエラーは処理できず、アプリは常にクラッシュする。

正解：B

DatabaseException を捕捉する専用の例外ハンドラーを置くと、制約違反（例：『重複した名前』）を具体的にフィードバックできます。汎用の All Exceptions はセーフティネットとして併用します（ローコードのハンドラーで行い、C# ではありません）。

Q22

複雑な JSON オブジェクトを入力として受け取る REST API を公開します。入力パラメータをどう定義しますか？

  • A入力パラメータの 'Data Type' を JSON レイアウトに合った 'Structure' に設定する。
  • B'Data Type' を Text にして手動で解析する。
  • C'Binary Data' を入力に使う。
  • DREST API は Integer や Text などの簡単な型しか受け付けない。

正解：A

JSON レイアウトに一致する Structure を定義し、それを API 入力に設定すると、プラットフォームが受信 JSON ボディを自動的にその変数にデシリアライズします（手動パース不要）。

Q24

リアクティブ Web アプリで、『クライアント変数』はどこで使用できますか？

  • AAggregates のソースとして。
  • Bサーバーアクションのみ。
  • Cタイマーを含むすべての場所で。
  • Dクライアントアクション、画面、ブロック（クライアント側のロジック）で。

正解：D

クライアント変数はブラウザのスコープに存在し、クライアント側ロジックで使えます。サーバーアクションやタイマーはサーバー側で動くため直接アクセスできず、必要なら入力パラメータとして渡します（Aggregate のフィルタには使えますがソースではありません）。

Q25

'Order' レコードの 'List' があり、全注文の 'TotalAmount' の合計を計算したい。どうしますか？

  • A'List' の 'Length' プロパティを使う。
  • B'ListUtils' Forge コンポーネントを使う。
  • C'For Each' でリストをループし、ローカル変数に値を蓄積する。
  • D'List' ウィジェットの 'Sum' プロパティを使う。
  • EOutSystems ではリストの合計はできない。

正解：C

データが DB にあるなら Aggregate の SUM が高速ですが、すでにメモリ内のリスト変数（例：API 取得結果）なら For Each でループして合計を蓄積します。Length は行数、リストに Sum プロパティはありません。
