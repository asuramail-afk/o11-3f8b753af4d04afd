Q1

Service Studio でエンティティを削除して公開すると、データベースのデータはどうなりますか？

  • Aデータは CSV ファイルにアーカイブされる。
  • Bテーブルは DB に残るが、OutSystems はそれをモジュールから切り離す（ソフト削除／デタッチ）。
  • Cテーブルとその全データが即座にドロップ（ハード削除）される。
  • D手動でテーブルを削除しない限り、公開は失敗する。

正解：B

Service Studio で Entity を削除してもメタデータ定義が消えるだけで、公開時にデータベースへ DROP TABLE は実行されません。テーブルとデータは DB に残ったままモジュールから切り離される（デタッチ）ため、誤削除からデータが保護されます。物理的に削除したい場合は DbCleaner API（Entity_DropTable）を使います。

Q2

'ExcelToRecordList' で、Excel の列名が 'Phone Number'（空白あり）、Structure 属性は 'PhoneNumber'（空白なし）です。自動的にマッピングされますか？

  • Aはい、'Automap' ボタンを使えば自動的にマッピングされる。
  • Bはい、両方が3番目の列ならマッピングされる。
  • Cはい、OutSystems は空白を無視してマッピングする。
  • Dいいえ。名前は正確に一致する必要がある（通常は大文字小文字は無視だが空白は重要）。属性名かヘッダー名を一致させる。

正解：D

ExcelToRecordList のマッピングは列の位置ではなく名前ベースで行われます。大文字小文字は無視されますが空白は区別されるため、'Phone Number' は 'PhoneNumber' と一致せず、その列は無視されます。Excel のヘッダー名か Structure の属性名を一致させる必要があります。

Q3

ダッシュボード画面で、ユーザーがページを更新しなくても5分ごとに自動でデータをリフレッシュしたい。どう実装しますか？

  • AContainer を画面にドラッグし、OnReady クライアントアクションで JavaScript の setInterval を使ってクライアントアクションをトリガーし、Aggregateをリフレッシュする。
  • BLogic タブの 'Timer' プロセスを使う。
  • CAggregate の 'Auto Refresh' プロパティを使う。
  • D'OnRender' イベントでリフレッシュをループさせる。

正解：A

Reactive アプリで定期的な自動リフレッシュを実現する標準の方法は、JavaScript の setInterval で一定間隔ごとに Client Action を呼び出し、その中で Aggregate をリフレッシュすることです。メモリリーク防止のため OnDestroy で clearInterval するのを忘れないでください。Timer はサーバー側のバッチ処理用で画面の更新には使えず、OnRender でのリフレッシュは無限ループになります。

Q4

ブロック 'UserInfo' が入力パラメータ 'UserId' を受け取り、内部の Aggregate 'GetProfile' を UserId でフィルタしています。親画面で UserId が変わると、この Aggregate はどうなりますか？

  • A自動的にはリフレッシュされない。'OnParametersChanged' で明示的にリフレッシュする必要がある。
  • Bブロックが破棄されて再作成される。
  • Cアプリケーションがエラーをスローする。
  • DAggregate は自動的にリフレッシュされる。

正解：A

Reactive では、Web Block の入力パラメータが変わっても、それを使っている Aggregate は自動では再実行されません。OnParametersChanged ライフサイクルイベントで明示的に Aggregate をリフレッシュする必要があります。これは Traditional Web との重要な違いです。

Q5

Logic Flow でコードにコメントを付ける有効な方法はどれですか？

  • Aノードの名前を // で始める。
  • Bフローエディタにはコメントを追加できない。
  • C背景を右クリックして 'ノートを追加' を選ぶ。
  • Dツールボックスから 'Comment' ツールをフローにドラッグする。

正解：D

ツールボックスの Comment ツールをフローのキャンバスにドラッグすると、テキストの注釈を直接配置できます。複雑なロジックの説明や、関連するノードの視覚的なグループ化に役立ちます。

Q6

画面が非常に遅く、Preparation/OnInitialize の Aggregate が過剰にデータを取得していました。一般的な最適化は？

  • Aサーバーのタイムアウトを増やす。
  • Bページネーション（Max Records）を実装し、取得する属性を最小限にする（使う列だけ取得）。
  • C静的エンティティを使う。
  • Dロジックをクライアント側に移動する。

正解：B

パフォーマンス最適化の基本は取得データを減らすことです。Max Records で行数を制限（ページネーション）し、実際に使う属性だけを取得します。Aggregate は使用されている属性を自動で最適化しますが、取得行数は開発者が Max Records で管理する必要があります。

Q7

『パーソナル環境（Personal Environment）』とは何ですか？

  • Aアプリケーションのテーマ。
  • B企業環境内のサンドボックス。
  • C学習やプロトタイピングのために個人開発者に提供される無料のクラウドベースの OutSystems 環境。
  • Dラップトップにインストールされたローカルの OutSystems。

正解：C

パーソナル環境は、OutSystems がクラウドでホスティングする無料のフルスタック環境（プラットフォームサーバー＋データベース）で、個人開発者が学習やプロトタイピングに使えます。自前でサーバーをインストールする必要はありませんが、容量やスリープなどの制限があります。

Q8

『Anonymous』ユーザーの『ロール（Role）』は何ですか？

  • Aログインしていない任意のユーザーを表す。
  • Bログインしているが名前のないユーザー。
  • C管理者（Administrator）ロール。
  • Dロボット専用のロール。

正解：A

Anonymous ロールはログインしていない任意のユーザーを表します。画面のセキュリティ設定で Anonymous をチェックすると、認証なしでアクセスできるパブリック画面（ランディングページやログイン画面など）を作成でき、チェックしない場合は未認証ユーザーはログイン画面へリダイレクトされます。

Q9

ユーザーが Table ウィジェットの行をクリックしたときにトリガーされるイベントは？

  • A行の設定次第。通常は行や特定のセル／リンクに 'On Click' イベントハンドラーを追加する。
  • BOnRowClick
  • COnSelect
  • DOnChange

正解：A

Table には既定の『行クリック』イベントは存在しません。行やその中のセル・リンク・コンテナに On Click イベントハンドラーを明示的に追加し、Client Action にバインドする必要があります。

Q10

ライブラリモジュールのサーバーアクションの 'Public' プロパティを 'Yes' にする意味は？

  • Aこのアクションは、環境内の他のモジュールから参照して使用できる。
  • Bこのアクションは、同じアプリ内のモジュールからのみ使用できる。
  • Cこのアクションは、外部ユーザーから匿名で呼び出せる。
  • Dこのアクションは自動的に REST API になる。

正解：A

Public プロパティを Yes にすると、その Server Action は環境内の他のモジュールから参照（依存関係として追加）して利用できるようになります。ただしこれはモジュール間の公開であり、外部への REST 公開には別途 REST API の設定が必要です。

Q11

Service Studio の 'TrueChange' エンジンの役割は？

  • Aコードをリアルタイムで分析し、公開前にエラー・警告・パフォーマンスの問題を検出する。
  • Bアプリケーションのバージョン管理を行う。
  • Cバグを自動的に修正する。
  • DSQL クエリを最適化する。

正解：A

TrueChange はバックグラウンドで常時コードを分析するコンパイラー兼バリデーターで、エラー・警告・パフォーマンス上の問題を公開前に検出して即時フィードバックします。型の不一致などの壊れたコードは公開できません。バグの自動修正や SQL の最適化は行いません。

Q12

'Users' アプリで新しいユーザーを作成するとき、必須のユニーク識別子のフィールドは？

  • AEmail（メールアドレス）
  • BPassword（パスワード）
  • CUsername（ユーザー名）
  • DFull Name（フルネーム）

正解：C

標準の User Entity では、ログインに使用する Username がユニークかつ必須の識別子です。Email もユニークにすることが多いですが、厳格なユニーク制約が課されているのは Username です。

Q13

'Order' エンティティの 'CreatedBy' 属性から、作成したユーザーの名前を Aggregate で取得するには？

  • A式内で 'GetUserName' 関数を使う。
  • BSystem エンティティとは結合できない。
  • CSQL クエリを作成する。
  • D'User' エンティティを Aggregate のソースに追加する。プラットフォームが結合条件（Order.CreatedBy = User.Id）を自動推測する。

正解：D

User システムエンティティを Aggregate のソースに追加すると、CreatedBy が User Identifier 型であることからプラットフォームが外部キー関係を検出し、Order.CreatedBy = User.Id の Join を自動作成します。単一クエリで取得できるため効率的で、式で1行ずつ取得する方法のような N+1 問題を避けられます。

Q14

エンティティ属性の『必須』(Is Mandatory) プロパティは何に使われますか？

  • A画面上の UI バリデーションだけに影響する。
  • B属性をプライマリキーにする。
  • Cデータを暗号化する。
  • Dデータベース制約（NOT NULL）と OutSystems のバリデーションを強制し、この値なしではレコードを保存できなくする。

正解：D

Is Mandatory を Yes にすると、データベースに NOT NULL 制約が適用されると同時に、画面上の入力も自動的に必須としてバリデーションされます。UI だけでなく DB レベルでもデータ整合性が守られるため、この値なしではレコードを保存できません。

Display Name は環境セレクタや管理コンソールで環境を識別するためのラベル（例：『本番』『UAT』）で、技術的な動作には影響しません。『開発環境だと思ったら本番だった』といった操作ミスの防止に役立ちます。

Q16

入力フィールドに有効なメールアドレスだけを受け入れたい。最も速く組み込みの方法は？

  • AOnChange イベントで正規表現を使う。
  • B組み込みのメールバリデーションは存在しない。
  • CInput ウィジェットの 'Input Type' プロパティを 'Email' に設定する。

Input ウィジェットの Input Type を Email に設定するのが最も速い組み込みの方法です。HTML5 のネイティブなクライアント側検証が追加ロジックなしで働き、モバイルではメール入力に適したキーボードも自動で表示されます。

Q17

公開時に『モジュール A は署名 Y のアクション X を期待するが、モジュール B は署名 Z を提供』というリファレンス/互換性エラーが出ました。どう修正しますか？

  • Aモジュール A を削除して再作成する。
  • BService Center のサービスを再起動する。
  • Cモジュール B（プロデューサ）を開いて変更を元に戻す。
  • Dモジュール A（コンシューマ）を開き『依存関係の管理』で B への参照をリフレッシュしてから A を公開する。

正解：D

このエラーは、コンシューマ（モジュール A）がプロデューサ（モジュール B）の古い定義を参照していることが原因です。A を開いて『依存関係の管理』で B への参照をリフレッシュし、A を再公開すれば新しい署名が反映されます。プロデューサの変更を元に戻すのは開発の巻き戻しであり、正しい対処ではありません。

Q18

モバイル対応で、内容を縦に並べ、小さな画面で自動的に積み重なる（スタックする）配置に最適なウィジェットは？

  • A'Columns' ウィジェット（Columns2、Columns3 など）または 'Adaptive Columns'。
  • B'Table' ウィジェット。
  • C固定幅の 'Container' ウィジェット。
  • D'If' ウィジェット。

正解：A

OutSystems UI の Columns ウィジェットはレスポンシブに設計されており、デスクトップでは横並び、モバイルやタブレットでは自動的に縦に積み重なります。Table は横スクロールが必要になりがちで、固定幅の Container はレスポンシブ性を損ないます。

Q19

'Feedback Message' クライアントアクションの目的は？

  • A一時的な通知（トースト）をユーザーに表示する（例：『成功』『エラー』）。
  • Bデータベースにログを書き込む。
  • C管理者にメールを送信する。
  • Dチャットウィンドウを開く。

正解：A

Feedback Message は、操作結果（『保存しました』『エラーが発生しました』など）をトースト形式で一時的に表示する標準の手段です。ユーザーの操作を妨げず、自動的に消えます。ログの書き込みには LogMessage を使います。

Q20

高精度の小数を含む複雑な数学計算を行います。どの変数データ型を使うべきですか？

  • Aテキスト（Text）
  • Bブーリアン（Boolean）
  • C整数（Integer）
  • D小数（Decimal）

正解：D

Decimal は金額計算や科学計算などの高精度な小数演算向けに設計されており、浮動小数点で起きがちな丸め誤差を避けられます。Integer は小数を扱えず、Text や Boolean は計算に適しません。

Q21

ロジックフローでレコードを更新中、DB 制約違反で更新が失敗し DB 例外がスローされます。このシナリオをどう処理しますか？

  • A'If' ノードで DB がオンラインかどうかを確認する。
  • Bフローに例外ハンドラーを追加し、例外タイプを 'DatabaseException' に設定してフォールバックのロジック（例：メッセージ表示）を実装する。
  • CC# コードの 'Try-Catch' ブロックで更新を囲む。
  • Dデータベースエラーは処理できず、アプリは常にクラッシュする。

正解：B

フローに Exception Handler を追加し、例外タイプを DatabaseException に設定するのが正しい方法です。制約違反などの DB エラーを捕捉して、メッセージ表示などのフォールバック処理をローコードで実装できます。汎用の All Exceptions はセーフティネットとして併用します。C# のコードを書く必要はありません。

Q22

複雑な JSON オブジェクトを入力として受け取る REST API を公開します。入力パラメータをどう定義しますか？

  • A入力パラメータの 'Data Type' を JSON レイアウトに合った 'Structure' に設定する。
  • B'Data Type' を Text にして手動で解析する。
  • C'Binary Data' を入力に使う。
  • DREST API は Integer や Text などの簡単な型しか受け付けない。

正解：A

JSON のレイアウトに一致する Structure を定義し、入力パラメータのデータ型に設定します。受信した JSON ボディはプラットフォームが自動的にその Structure へデシリアライズするため、手動でのパースは不要です。

Q23

モジュールは、画面・ロジック・データをカプセル化した開発および公開の単位で、OML ファイルとして Service Studio で開発され、個別に公開できます。階層は『アプリケーション → モジュール』です。

Q24

リアクティブ Web アプリで、『クライアント変数』はどこで使用できますか？

  • AAggregates のソースとして。
  • Bサーバーアクションのみ。
  • Cタイマーを含むすべての場所で。
  • Dクライアントアクション、画面、ブロック（クライアント側のロジック）で。

Q25

'Order' レコードの 'List' があり、全注文の 'TotalAmount' の合計を計算したい。どうしますか？

  • A'List' の 'Length' プロパティを使う。
  • B'ListUtils' Forge コンポーネントを使う。
  • C'For Each' でリストをループし、ローカル変数に値を蓄積する。
  • D'List' ウィジェットの 'Sum' プロパティを使う。
  • EOutSystems ではリストの合計はできない。

正解：C

すでにメモリ上にあるリスト変数（API の取得結果など）の合計は、For Each でループしながら Local Variable に値を蓄積して計算します。データがまだ DB にあるなら Aggregate の SUM 集計の方が効率的です。Length は行数を返すだけで、List に Sum プロパティは存在しません。
