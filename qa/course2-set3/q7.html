Q7🖼 画像あり

データモデルで Description は1500文字、HighlightedPicture は binary です。Description の部分一致検索のクエリ性能を上げる変更は？

  • AProperty を2分割（一方に基本詳細、もう一方に残り）する。
  • BHasParking/HasGarage を int 属性に置き換える。
  • CDescription 属性にインデックスを作成する。
  • DHighlightedPicture と HighlightedLabel を持つ新しい拡張エンティティを作成する。

正解：D

サイズの大きい binary 属性（HighlightedPicture）を別の拡張 Entity に分離すると、本体テーブルの行が軽くなり、Description の部分一致検索のクエリ性能が向上します。なお部分一致（LIKE）検索ではインデックス作成（C）は効果がありません。
