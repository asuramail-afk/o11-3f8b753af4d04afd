Q18🖼 画像あり

複数アプリを管理する企業がデプロイに苦労。一部は共通プロデューサーの機能を消費。左＝元、右＝新しい解決策。正しい選択肢は？

  • A元は Customer アプリの公開 Server Actions を消費して密結合でデプロイ困難だった。新解決策は同機能を Service Actions で公開し疎結合化するが、複数リクエスト/トランザクション処理の追加ロジックが必要でコストが増える。
  • B違いは、元が Server Actions を公開、新が Web サービスを公開する点。Web サービスで独立リリースでき、Customer 変更時にコンシューマ更新が不要になる。
  • C元は Customer のスクリーンを公開して密結合だった。新は External URLs で公開し疎結合化。この変更に欠点はなく独立リリースでき価値が向上する。
  • Dすべての選択肢が正しい。

正解：A

元の構成は Customer アプリの公開 Server Actions を消費する強い依存で、密結合となりデプロイが困難でした。Service Actions に変更すると弱い依存となり疎結合化・独立リリースが可能になります。ただし Service Actions は呼び出し元とは別のリクエスト/トランザクションで実行されるため、失敗時の整合性処理など追加ロジックが必要になりコストが増える、という A の記述が正確です。
