【シナリオ】外部データベースの顧客情報を更新する POST REST API メソッドについて。インテグレーションチームから「PhoneNumber 属性は null にできません」というエラーの解決を依頼されました。API が Name と生年月日を正しく受け取っていることは確認済みで、PhoneNumber（電話番号）はエンドユーザーが必ず入力する必須フィールドではありません。このエラーをより良く解決する方法は？

  • A「PhoneNumber」入力パラメータの「Is Mandatory（必須）」プロパティを「Yes」に設定する。
  • BREST API の「Send Default Value（デフォルト値を送信）」プロパティを「Yes」に設定する。
  • C「PhoneNumber」入力パラメータの「Default Value（デフォルト値）」を、何も渡されなかったときに使う標準の電話番号に設定する。
  • D「OnBeforeRequest」コールバックを使い、何も渡されなかったときに「PhoneNumber」へ標準の電話番号を設定する。

正解：B

PhoneNumber は任意入力のため、値が渡されないと API に null が送信され「null にできません」エラーになります。REST API の Send Default Value プロパティを Yes に設定すると、値が渡されなかった場合に入力パラメータの既定値が送信され、null の送信を回避できます。任意項目を Is Mandatory=Yes にするのは要件に反します。
