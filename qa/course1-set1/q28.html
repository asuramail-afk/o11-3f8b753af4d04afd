Q28🖼 画像あり

図のアクションはスーパーマーケットの請求書 Excel を処理し Invoice/InvoiceDetails に挿入しますが非効率でタイムアウトします。効率化しタイムアウトを防ぐ選択肢は？

  • A一時テーブルを作りタイマーで非同期処理。各反復末尾で CommitTransaction を呼ぶ。
  • BGetProduct/GetWarehouse のAggregateをキャッシュしてタイムアウトを防ぐ。
  • Cタイマーで非同期挿入し、Aggregateをキャッシュ可能な単一クエリに変換する。
  • D一時テーブルを作り、タイムアウト管理付きタイマーで非同期処理。GetProduct/GetWarehouse Aggregateは別アクションに分離し結果をキャッシュする。

正解：D

一時テーブルに取り込んでからタイムアウト管理付きの Timer で非同期処理し、繰り返し呼ばれる GetProduct / GetWarehouse の Aggregate は別の Server Action に切り出して結果をキャッシュするのが最も効率的です。ユーザーを待たせず、重複クエリも排除できます。
