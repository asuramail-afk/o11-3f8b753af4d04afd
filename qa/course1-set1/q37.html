GetAllApplications アクションはアプリ一覧を返し他アクションから使われます。GetApplications Aggregateを直接使うのと GetAllApplications を呼ぶのとで、直接使う良い理由は？

  • A直接Aggregateを呼ぶと、アクションは必要に応じ結果をキャッシュでき、キャッシュはAggregateにのみ適用される。
  • BGetAllApplications 呼び出しをAggregateの直接実行に置き換える理由はない。
  • C直接Aggregateを呼ぶとアクション間の独立性が促進され、個別に進化させやすい。
  • D直接Aggregateを呼ぶと、プラットフォームが呼び出しごとに使用内容に基づきAggregateと出力を最適化できる。

正解：D

Aggregate を画面やアクションで直接使うと、プラットフォームは呼び出し箇所ごとに実際に使用される属性だけを取得するようクエリを自動最適化します。Aggregate を Server Action の中に包むと、呼び出し側がどの列を使うか分からないため全カラムを取得してしまいます。なお A は誤りで、キャッシュは Server Action にも設定できます。
