Q1

アプリケーション全体のプライマリーカラーを変更したい。最も効率的な方法は？

  • A'.btn-primary' クラスを各画面でオーバーライドする。
  • Bすべてのボタンの色を手動で変更する。
  • C新しいテーマを作成する。
  • Dアプリケーションテーマ（CSS）内の CSS 変数（例：--color-primary）の値を更新する。

正解：D

最新の OutSystems テーマは CSS 変数（カスタムプロパティ）を使います。テーマモジュール内で --color-primary 等の定義を変えれば、全ウィジェット・画面に即座に色が反映されます。

Q2

クライアントアクションのロジックフローを 'Admin' ロールのユーザーだけに制限したい。どう確認しますか？

  • Aフロー冒頭の 'If' ノードで 'CheckAdminRole(GetUserId())' 関数を使い、False なら例外を投げるかリダイレクトする。
  • Bクライアントアクションの 'Roles' プロパティを設定する。
  • C'Authorize' ウィジェットを使う。
  • Dクライアントアクションを制限することはできない。

正解：A

アクションに 'Roles' プロパティはありません（画面のみ）。ロジック内で Check<Role>() を明示的にチェックします。クライアント側チェックは UX 向上用で、真のセキュリティはサーバーアクション内でもロールを検証します。

Q3

'On Session Start' システムイベントは何に使われますか？

  • Aデータベースキャッシュをクリアするため。
  • Bユーザーをログインページにリダイレクトするため。
  • Cすべてのリクエスト時にユーザーがログインしているか確認するため。
  • Dユーザーが最初にアプリにアクセスした（新しいセッションを開始した）ときに特定のロジックを実行するため（例：セッション変数の初期化）。

正解：D

On Session Start は新しいセッションコンテキストが作成されたときにサーバー側で一度だけ実行される初期化フックで、既定のユーザー設定やセッション開始の監査などに使います（リクエストごとではありません）。

Q4

データベーステーブルの有効な 'トリガー' はどれですか？

  • A画面のロード前（Before Screen Load）
  • B選択後（After Select）
  • C挿入後（After Insert）
  • Dクリック後（On Click）

正解：C

DB トリガーはレコードの挿入・更新・削除の前後に実行できます（After Insert など）。読み取り（Select）では発火せず、Before Screen Load / On Click は UI イベントです。

Q5

'Order' エンティティに 'StatusId' 外部キーがあります。'StatusId' の 'Delete Rule' を 'Ignore' にした場合、'Status' レコードを削除するとどうなりますか？

  • Aデータベースが削除を防ぎ、エラーを返す。
  • B'Status' が削除され、'Order' はそのままで存在しない 'StatusId' を指す（孤児になる）。
  • C'Order' も削除される。
  • D'Order' の 'StatusId' が Null に設定される。

正解：B

'Ignore' は参照整合性チェックを無効にします。親（Status）を削除しても子（Order）は残り、存在しない ID を指し続けます（整合性は崩れるが削除は可能）。防止は 'Protect'、カスケードは 'Delete'、Null 化は 'Nullify'。

Q6

次のうち、サイトプロパティの有効なデータ型では「ない」ものは？

  • Aエンティティ識別子
  • Bテキスト
  • C整数
  • Dレコード（またはリスト）

正解：D

サイトプロパティは単純な設定値（Text/Integer/Boolean/Date など）用で、レコードやリストなどの複雑な型は格納できません（キャッシュに載るためメモリ性能にも悪影響）。

Q7

Service Center で 'モジュール' を削除するとどうなりますか？

  • Aそれを使用するすべてのアプリケーションを削除する。
  • Bサーバーをフォーマットする。
  • C関連付けられたデータベースのテーブルを削除する。
  • Dモジュールはソフトデリートとしてマークされ、環境の実行中アプリからは消えるが、データベースのメンテナンスで復元/完全削除できる。

正解：D

モジュール（コード）を削除しても、対応するデータベースのテーブル/データは削除されません（『データは残る』ルール）。Service Center はモジュールをソフトデリートします。

Q8

Aggregate で 'Category' でレコードをグループ化しています。さらに 'Price' 属性も出力に表示したい場合、何をする必要がありますか？

  • A'Price' にも 'Group By' を使う。
  • B'Price' のために別の Aggregate を作成する。
  • C'Price' 属性に集計関数（Sum / Average / Max など）を適用する。
  • Dそのまま 'Price' 属性を出力リストにドラッグする。

正解：C

属性でグループ化すると、出力はグループ化列と、その他の列に適用した集計関数（Sum / Average / Max / Count 等）の結果になります。生の明細はそのままでは出力できません。

Q9

Service Studio の 'Personal Area' とは何ですか？

※重要：'Personal Area'（Run in Personal Area）は従来（Traditional）Web の機能で、Reactive Web Apps ではサポートされていません。設問が Reactive の文脈なら『サポートされていない』が正解になり得る点に注意。

  • Aデスクトップ上のフォルダ。
  • Bプライベートなデータベース。
  • Cテーマをカスタマイズするエリア。
  • D他の開発者に影響を与えず、サーバー上のプライベートなサンドボックスでモジュールを実行・デバッグできる機能。

正解：D

'Run in Personal Area' は変更を隔離して実行・デバッグする機能です。

Q10

エンティティ属性（整数型）の 'Auto-Number' プロパティの目的は？

  • A数字にカンマを付けてフォーマットする。
  • B自動的に合計を計算する。
  • C属性をアイデンティティ列（自動増分）にし、DB が各新規レコードに一意の連番を自動割り当てするようにする。
  • D数字が常に正の数になるようにする。

正解：C

Auto-Number はアイデンティティ列（自動増分）を設定し、DB が 1, 2, 3… と一意の ID を自動採番します。新しいエンティティの 'Id' 属性は既定で Auto-Number=Yes です。

Q11

'OnNotify' イベントを持つブロックがあります。なぜ標準の Reactive Web 開発で『非推奨（Deprecated）』とされているのですか？

  • A遅すぎるから。
  • BReactive Web は入力パラメータを持つ『Events』を使い、従来 Web の緩い 'Notify'（メッセージ文字列）パターンを、厳密に型付けされた現代的な仕組みで置き換えているから。
  • Cモバイルアプリ専用だから。
  • Dブロックが親と通信できなくなったため削除された。

正解：B

'Notify' は一般的なテキスト文字列を渡していましたが、'Events' は型付けされたデータ（Integer・Record 等）を入力パラメータとして渡せて安全でクリーンです。Reactive/モバイルでは常に Events を使います。

Q12

OutSystems における 'Structure' とは何ですか？

  • A論理フロー。
  • B画面レイアウト。
  • Cデータベースのテーブル。
  • Dメモリ内に定義される、属性のコレクションから成るカスタムデータ型で、データベースには保存されない。

正解：D

Structure はクラスや DTO のようなもので、API・複合変数・カスタム戻り値のデータの形を定義します。エンティティ（＝DB テーブル）と異なり、テーブルを作らずメモリ内のオブジェクトに相当します。

Q13

特定のページにアクセスしようとしたときに『アクセス拒否（Access Denied）』が表示される。最も考えられる原因は？

  • Aその画面が削除されている。
  • Bユーザーが、画面の 'Roles' タブでチェックされているいずれのロール（Roles）も持っていない。
  • Cユーザーのパスワードが期限切れになっている。
  • Dその画面が 'Anonymous' のみに設定されている。

正解：B

システムは User.Roles と Screen.Roles の共通部分をチェックし、空なら 'Access Denied' を返します（画面削除は 404、パスワード期限切れはログイン要求で別問題）。

Q14

'List' 型の変数にレコードが含まれているかを確認したい。最も効率的な方法は？

  • AListFilter(MyList, ...)
  • BListAny(MyList)
  • Cリストをループする
  • DMyList.Empty（ランタイムプロパティ）または List.Length = 0

正解：D

リストが空かどうかは、組み込みの Empty（Boolean）プロパティや Length で即座に判定できます。ListFilter やループはやり過ぎ/低速です。

Q15

Container に OnClick が設定され、その中に Button があります。そのボタンをクリックすると Container の OnClick も発火しますか？

  • Aいいえ、ボタンがコンテナを覆っているから。
  • Bいいえ、OutSystems は既定でバブリングを防止する。
  • Cはい、ただしボタンが無効（disabled）の場合のみ。
  • Dはい、イベントは DOM 内でバブル（伝播）します（明示的に伝播を停止しない限り）。

正解：D

標準の HTML/JavaScript（Reactive も同様）では、子要素のクリックで親のクリックリスナーも発火します。防ぐには JavaScript で e.stopPropagation() を使います。

Q16

ユーザー生成の HTML スニペットを Expression ウィジェットに表示していますが、既定では HTML タグがテキスト（例：<b>Bold</b>）として表示されます。ブラウザに書式をレンダリングさせるには？

  • A'SanitizeHtml' 関数を使う。
  • B'FormatHtml' 関数を使う。
  • CExpression を 'Type = HTML' のコンテナでラップする。
  • DExpression の 'Escape Content' プロパティを 'No' に設定する。

正解：D

既定では Expression は特殊文字をエスケープします。'Escape Content = No' にすると生の HTML をレンダリングします（※必ずサニタイズ済みの安全なコンテンツにのみ適用）。

Q17

Development から Quality へ Solution を展開中に『Missing Configuration（設定なし）』警告が出ました。通常これは何を意味しますか？

  • A対象環境（Quality）のサイトプロパティまたは REST API の URL に値が設定されていない。
  • Bデータベースがいっぱいになっている。
  • Cコードを手動でコンパイルする必要がある。
  • Dモジュールが不足している。

正解：A

LifeTime は新しい設定パラメータ（サイトプロパティ等）を検出し、展開完了前に新環境で値を設定するよう促します。これにより『自分の環境では動くのに』というエラーを防ぎます。

Q18

REST API から返るデータを格納する 'Structure' を定義しています。API は 'user_address' という JSON フィールドを返します。これを 'Address' という Structure 属性にマッピングするには？

  • AREST 統合ウィザードの '自動マッピング（Automap）' 機能を使う。
  • B正確に一致しなければならないのでマッピングできない。
  • CStructure 属性 'Address' の 'JSON Name' プロパティを 'user_address' に設定する。
  • DStructure 属性の名前を 'user_address' にする必要がある。

正解：C

'JSON Name' プロパティで、内部の命名規則（PascalCase：Address）と外部 API の命名規則（snake_case：user_address）を切り離せます。属性名自体を変えるのは命名規約を壊すため非推奨です。

Q19

リアクティブ Web アプリで 'サーバーアクション' を '関数' に設定しています。この関数を画面上の '式（Expression）' で直接呼び出せますか？

  • Aはい、自動的に動作する。
  • Bはい、しかし遅くなる。
  • Cいいえ。サーバーアクション（関数を含む）はクライアント側の式で直接呼び出せない。
  • D'Public' としてマークされている場合のみ可能。

正解：C

クライアント側の式（HTML 内）は即座に解決される必要があり、サーバーへのラウンドトリップを待てません。値は事前に Data Action や Aggregate で取得し、その結果に結び付けます（式で使えるのはクライアントアクション/関数や組み込み関数）。

Q20

データベースの 'バイナリデータ（Binary Data）' 属性の最大サイズはどれですか？

  • A1MB
  • B500KB
  • Cデータベースエンジンによるが、実質的にはデータベースの最大 BLOB サイズ（例：2GB）に制限される
  • D10MB

正解：C

OutSystems は Binary Data を BLOB（Oracle）や VARBINARY(MAX)（SQL Server）にマッピングし、大きなファイルも保存できます。ただし性能面では、外部ストレージ（S3/Blob 等）に置きメタデータだけ DB に持つ方が推奨されます。
