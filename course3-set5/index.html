Q1

アプリケーション全体のプライマリーカラーを変更したい。最も効率的な方法は？

  • A'.btn-primary' クラスを各画面でオーバーライドする。
  • Bすべてのボタンの色を手動で変更する。
  • C新しいテーマを作成する。
  • Dアプリケーションテーマ（CSS）内の CSS 変数（例：--color-primary）の値を更新する。

正解：D

最新の OutSystems テーマは CSS 変数（カスタムプロパティ）で色を管理しています。テーマの CSS 内で --color-primary などの値を 1 か所変更するだけで、アプリ全体のウィジェットや画面に一括で反映されるため、画面ごとのオーバーライドや手動変更より圧倒的に効率的です。

Q2

クライアントアクションのロジックフローを 'Admin' ロールのユーザーだけに制限したい。どう確認しますか？

  • Aフロー冒頭の 'If' ノードで 'CheckAdminRole(GetUserId())' 関数を使い、False なら例外を投げるかリダイレクトする。
  • Bクライアントアクションの 'Roles' プロパティを設定する。
  • C'Authorize' ウィジェットを使う。
  • Dクライアントアクションを制限することはできない。

正解：A

Roles プロパティを持つのは画面だけで、アクションには存在しません。そのためロジック内で Check<ロール名>Role() 関数を使い、False なら例外やリダイレクトで制御します。なお Client Action でのチェックは UX 向上のためのもので、真のセキュリティ担保には Server Action 側でも必ずロールを検証する必要があります。

Q3

'On Session Start' システムイベントは何に使われますか？

  • Aデータベースキャッシュをクリアするため。
  • Bユーザーをログインページにリダイレクトするため。
  • Cすべてのリクエスト時にユーザーがログインしているか確認するため。
  • Dユーザーが最初にアプリにアクセスした（新しいセッションを開始した）ときに特定のロジックを実行するため（例：セッション変数の初期化）。

正解：D

On Session Start は、新しいセッションが作成されたタイミングでサーバー側で一度だけ実行される初期化フックです。Session Variable の初期化や既定のユーザー設定、セッション開始の監査ログなどに使います。リクエストごとに実行されるわけではない点に注意してください。

Q4

データベーステーブルの有効な 'トリガー' はどれですか？

  • A画面のロード前（Before Screen Load）
  • B選択後（After Select）
  • C挿入後（After Insert）
  • Dクリック後（On Click）

正解：C

データベーストリガーはレコードの挿入・更新・削除の前後（After Insert など）に実行されるもので、読み取り（Select）では発火しません。Before Screen Load や On Click は UI 側のイベントであり、データベーストリガーではありません。

Q5

'Order' エンティティに 'StatusId' 外部キーがあります。'StatusId' の 'Delete Rule' を 'Ignore' にした場合、'Status' レコードを削除するとどうなりますか？

  • Aデータベースが削除を防ぎ、エラーを返す。
  • B'Status' が削除され、'Order' はそのままで存在しない 'StatusId' を指す（孤児になる）。
  • C'Order' も削除される。
  • D'Order' の 'StatusId' が Null に設定される。

正解：B

Delete Rule を Ignore にすると参照整合性チェックが行われなくなります。親の Status を削除しても子の Order はそのまま残り、存在しない StatusId を参照する孤児レコードになります。削除を防ぐのは Protect、子も一緒に削除するのは Delete、外部キーを Null にするのは Nullify です。

Q6

次のうち、サイトプロパティの有効なデータ型では「ない」ものは？

  • Aエンティティ識別子
  • Bテキスト
  • C整数
  • Dレコード（またはリスト）

正解：D

Site Property は Text・Integer・Boolean・Date などの単純な設定値を保持するためのもので、Record や List のような複合型は格納できません。値はキャッシュに載るため、複雑なデータを持たせる設計はそもそも想定されていません。

Q7

Service Center で 'モジュール' を削除するとどうなりますか？

  • Aそれを使用するすべてのアプリケーションを削除する。
  • Bサーバーをフォーマットする。
  • C関連付けられたデータベースのテーブルを削除する。
  • Dモジュールはソフトデリートとしてマークされ、環境の実行中アプリからは消えるが、データベースのメンテナンスで復元/完全削除できる。

正解：D

Service Center で Module を削除するとソフトデリートとして扱われ、環境の実行中アプリケーションからは見えなくなりますが、データベースメンテナンスから復元または完全削除が可能です。なお、Module を削除しても対応するデータベースのテーブルやデータは削除されません。

Q8

Aggregate で 'Category' でレコードをグループ化しています。さらに 'Price' 属性も出力に表示したい場合、何をする必要がありますか？

  • A'Price' にも 'Group By' を使う。
  • B'Price' のために別の Aggregate を作成する。
  • C'Price' 属性に集計関数（Sum / Average / Max など）を適用する。
  • Dそのまま 'Price' 属性を出力リストにドラッグする。

正解：C

Aggregate で Group By を使うと、出力はグループ化した列と、他の列に適用した集計関数の結果だけになります。そのため Price を出力に含めるには Sum・Average・Max などの集計関数を適用する必要があり、明細のままの値は出力できません。

Q9

Service Studio の 'Personal Area' とは何ですか？

※重要：'Personal Area'（Run in Personal Area）は従来（Traditional）Web の機能で、Reactive Web Apps ではサポートされていません。設問が Reactive の文脈なら『サポートされていない』が正解になり得る点に注意。

  • Aデスクトップ上のフォルダ。
  • Bプライベートなデータベース。
  • Cテーマをカスタマイズするエリア。
  • D他の開発者に影響を与えず、サーバー上のプライベートなサンドボックスでモジュールを実行・デバッグできる機能。

正解：D

Personal Area（Run in Personal Area）は、サーバー上のプライベートなサンドボックスで自分の変更だけを実行・デバッグできる機能です。パブリッシュ済みの共有バージョンに影響を与えないため、他の開発者の作業を妨げずに動作確認ができます。

Q10

エンティティ属性（整数型）の 'Auto-Number' プロパティの目的は？

  • A数字にカンマを付けてフォーマットする。
  • B自動的に合計を計算する。
  • C属性をアイデンティティ列（自動増分）にし、DB が各新規レコードに一意の連番を自動割り当てするようにする。
  • D数字が常に正の数になるようにする。

正解：C

Auto-Number は属性をアイデンティティ列（自動増分）にする設定で、新規レコードの挿入時にデータベースが 1, 2, 3… と一意の連番を自動採番します。新しく作成した Entity の Id 属性は既定で Auto-Number が Yes になっています。

Q11

'OnNotify' イベントを持つブロックがあります。なぜ標準の Reactive Web 開発で『非推奨（Deprecated）』とされているのですか？

  • A遅すぎるから。
  • BReactive Web は入力パラメータを持つ『Events』を使い、従来 Web の緩い 'Notify'（メッセージ文字列）パターンを、厳密に型付けされた現代的な仕組みで置き換えているから。
  • Cモバイルアプリ専用だから。
  • Dブロックが親と通信できなくなったため削除された。

正解：B

従来の Notify は単なるテキスト文字列しか渡せない緩い仕組みでした。Reactive Web の Events は Integer や Record などの型付きの入力パラメータを渡せる安全でクリーンな仕組みであり、これが Notify を置き換えたため OnNotify は非推奨となっています。Reactive／モバイルでは常に Events を使います。

Q12

OutSystems における 'Structure' とは何ですか？

  • A論理フロー。
  • B画面レイアウト。
  • Cデータベースのテーブル。
  • Dメモリ内に定義される、属性のコレクションから成るカスタムデータ型で、データベースには保存されない。

正解：D

Structure は属性の集まりとして定義するカスタムデータ型で、メモリ内でのみ使われ、データベースには保存されません。クラスや DTO に相当し、API のデータ形状や複合変数の定義に使います。データベーステーブルを作る Entity とはこの点で異なります。

Q13

特定のページにアクセスしようとしたときに『アクセス拒否（Access Denied）』が表示される。最も考えられる原因は？

  • Aその画面が削除されている。
  • Bユーザーが、画面の 'Roles' タブでチェックされているいずれのロール（Roles）も持っていない。
  • Cユーザーのパスワードが期限切れになっている。
  • Dその画面が 'Anonymous' のみに設定されている。

正解：B

画面アクセス時、システムはユーザーが持つロールと画面の Roles タブでチェックされたロールの共通部分を確認し、1 つも一致しなければ Access Denied を返します。画面が削除されていれば 404、パスワード期限切れならログイン要求となり、いずれも別の事象です。

Q14

'List' 型の変数にレコードが含まれているかを確認したい。最も効率的な方法は？

  • AListFilter(MyList, ...)
  • BListAny(MyList)
  • Cリストをループする
  • DMyList.Empty（ランタイムプロパティ）または List.Length = 0

正解：D

List が空かどうかは、組み込みの Empty プロパティ（Boolean）や Length = 0 で即座に判定できます。ListFilter やループでの走査はリスト全体を処理するため無駄が多く、単純な有無の確認には不適切です。

Q15

Container に OnClick が設定され、その中に Button があります。そのボタンをクリックすると Container の OnClick も発火しますか？

  • Aいいえ、ボタンがコンテナを覆っているから。
  • Bいいえ、OutSystems は既定でバブリングを防止する。
  • Cはい、ただしボタンが無効（disabled）の場合のみ。
  • Dはい、イベントは DOM 内でバブル（伝播）します（明示的に伝播を停止しない限り）。

正解：D

はい、発火します。Reactive を含む標準の HTML/JavaScript では、子要素のクリックイベントは DOM を親方向にバブル（伝播）するため、Button のクリックで Container の OnClick も実行されます。防ぎたい場合は JavaScript で e.stopPropagation() を呼び出します。

Q16

ユーザー生成の HTML スニペットを Expression ウィジェットに表示していますが、既定では HTML タグがテキスト（例：<b>Bold</b>）として表示されます。ブラウザに書式をレンダリングさせるには？

  • A'SanitizeHtml' 関数を使う。
  • B'FormatHtml' 関数を使う。
  • CExpression を 'Type = HTML' のコンテナでラップする。
  • DExpression の 'Escape Content' プロパティを 'No' に設定する。

正解：D

Expression は既定で特殊文字をエスケープするため、HTML タグがそのまま文字列として表示されます。Escape Content プロパティを No にすると生の HTML としてレンダリングされます。ただし XSS を防ぐため、必ずサニタイズ済みの安全なコンテンツにのみ使用してください。

Q17

Development から Quality へ Solution を展開中に『Missing Configuration（設定なし）』警告が出ました。通常これは何を意味しますか？

  • A対象環境（Quality）のサイトプロパティまたは REST API の URL に値が設定されていない。
  • Bデータベースがいっぱいになっている。
  • Cコードを手動でコンパイルする必要がある。
  • Dモジュールが不足している。

正解：A

Missing Configuration 警告は、対象環境（Quality）で Site Property や REST API の URL などの設定値が未設定であることを意味します。LifeTime が新しい設定パラメータを検出して展開前に値の設定を促すことで、環境差異による「自分の環境では動くのに」というエラーを防ぎます。

Q18

REST API から返るデータを格納する 'Structure' を定義しています。API は 'user_address' という JSON フィールドを返します。これを 'Address' という Structure 属性にマッピングするには？

  • AREST 統合ウィザードの '自動マッピング（Automap）' 機能を使う。
  • B正確に一致しなければならないのでマッピングできない。
  • CStructure 属性 'Address' の 'JSON Name' プロパティを 'user_address' に設定する。
  • DStructure 属性の名前を 'user_address' にする必要がある。

正解：C

Structure 属性の JSON Name プロパティに 'user_address' を設定すれば、内部の命名規則（PascalCase の Address）と外部 API の命名規則（snake_case の user_address）を切り離してマッピングできます。属性名自体を API に合わせて変えるのは命名規約を崩すため推奨されません。

Q19

リアクティブ Web アプリで 'サーバーアクション' を '関数' に設定しています。この関数を画面上の '式（Expression）' で直接呼び出せますか？

  • Aはい、自動的に動作する。
  • Bはい、しかし遅くなる。
  • Cいいえ。サーバーアクション（関数を含む）はクライアント側の式で直接呼び出せない。
  • D'Public' としてマークされている場合のみ可能。

正解：C

呼び出せません。画面上の式はクライアント側で即座に評価される必要があり、サーバーへのラウンドトリップを待てないため、Server Action は関数であっても式から直接呼び出せません。値は事前に Data Action や Aggregate で取得し、式ではその結果や Client Action の関数・組み込み関数を使います。

Q20

データベースの 'バイナリデータ（Binary Data）' 属性の最大サイズはどれですか？

  • A1MB
  • B500KB
  • Cデータベースエンジンによるが、実質的にはデータベースの最大 BLOB サイズ（例：2GB）に制限される
  • D10MB

正解：C

Binary Data 属性は Oracle では BLOB、SQL Server では VARBINARY(MAX) にマッピングされるため、上限は実質的にデータベースエンジンの最大 BLOB サイズ（例：2GB）です。ただし性能の観点からは、大きなファイルは S3 や Blob ストレージなどの外部に置き、DB にはメタデータのみ保存する構成が推奨されます。
