Q1

OutSystems でデータベースのリレーションシップ（関係）を定義する場所はどこですか？

  • AAggregate
  • Bプロセスタイマー
  • CWeb Block
  • D画面ロジック
  • Eサーバーアクション
  • Fエンティティデザイナー

正解：F

データベースのリレーションシップは、Entity デザイナーで外部キー（Reference 属性）を設定して定義します。1対1・1対多・多対多のいずれもここでモデル化します。Aggregate はデータの読み取り用であり、リレーションシップの定義は行いません。

Q2

OutSystems で要素をスタイル付けするにはどうしますか？

  • AStyle Class を割り当て、Theme で CSS を定義する
  • BRole を使用する
  • CAggregate の設定を使う
  • DJavaScript を表示する
  • ETimer を呼び出す
  • FEntity を使用する

正解：A

要素に Style Class を割り当て、その CSS を Theme に定義するのが推奨される方法です。スタイルを一元管理でき、アプリ全体で一貫した見た目を保てます。

Q3

入力値の検証ロジックはどこに配置すべきですか？

  • AJavaScript でバリデーションを行う
  • Bサーバーアクションに配置し、保存前に検証する
  • Cタイマーを使って保存後に検証する
  • D送信ボタンだけで検証を行う
  • E保存後に警告を表示する
  • F静的エンティティの検証

正解：B

データの整合性を守るには、保存前に Server Action（サーバー側）で検証するのが最も安全です。JavaScript などクライアント側だけの検証は、リクエストの改ざんによってバイパスされる可能性があります。

Q4

実行時のパフォーマンスを監視するために使用するツールはどれですか？

  • AService Center の Performance Monitoring タブ
  • BEntity デザイナー
  • CJavaScript デバッグ
  • DWeb Block エディター
  • EAggregate デザイナー
  • FCDT マネージャー

正解：A

実行時のパフォーマンスは Service Center の Performance Monitoring タブで監視します。画面のロード時間やロジックの実行時間などを追跡し、ボトルネックを特定できます。

Q5

画面間のナビゲーションを作成するために使用されるコンポーネントは何ですか？

  • ATimer
  • BWeb Block
  • CEntity
  • DLink または Button と Destination プロパティ
  • EAggregate
  • FAction

正解：D

Link や Button の Destination プロパティに遷移先の画面を設定することで、クリック操作による画面間のナビゲーションを実現します。

Q6

エンティティ属性を削除したときに何が起こりますか？

  • ATimer が壊れる
  • BWeb Block が失敗する
  • Cエンティティは変更されない
  • D対応する列がデータベースから削除される
  • ECDT が壊れる
  • FRole が削除される

正解：D

Entity の属性を削除すると、パブリッシュ時にデータベースの対応する列も削除され、スキーマが更新されます。列に格納されていたデータも失われる点に注意が必要です。

Q7

エンティティアクションは何をしますか？

  • ATimer を呼び出す
  • BStyle Sheet を使用する
  • CWeb Block を追加する
  • DCSS を適用する
  • EJavaScript を使用する
  • Fエンティティに対して作成・更新・削除（Create/Update/Delete）を行う

正解：F

Entity アクション（CreateXxx / UpdateXxx / DeleteXxx など）は、Entity のレコードに対する作成・更新・削除を行う、自動生成された書き込み用アクションです。

Q8

画面の入力パラメータはどこで設定しますか？

  • AOutput セクション
  • BStyle class
  • CTimer events
  • DAggregate のフィルタ
  • E画面の Input Parameters セクション
  • FRole マネージャー

正解：E

画面の入力パラメータは、画面の Input Parameters セクションで定義します。これにより、他の画面から遷移時に値を受け取れます。

Q9

ユーザーが権限なしで画面にアクセスしようとした場合、何が起きますか？

  • ATimer を表示する
  • B既定の『No Permissions（権限なし）』エラー画面にリダイレクトされる
  • Cメールを送信する
  • Dアクセスが許可される
  • EPreparation を実行する
  • Fログインページにリダイレクトされる

正解：B

必要なロールを持たないユーザーが画面にアクセスすると、既定の『No Permissions（権限なし）』エラー画面（Invalid Permissions 画面）にリダイレクトされます。アクセスが黙って許可されることはありません。

Q10

特定のユーザーだけがアクセスできる画面にするにはどうすればよいですか？

  • ATimer を使う
  • Bロールベースのアクセス制御（Role-based access control）を使う
  • CAction を追加する
  • DEntity を追加する
  • EAggregate を使う
  • FBlock を追加する

正解：B

画面にアクセスできるロールを設定するロールベースのアクセス制御を使います。指定したロールを持つユーザーだけがその画面にアクセスでき、それ以外は拒否されます。

Q11

レスポンシブな UI デザインを可能にするコンポーネントは何ですか？

  • ELayout / Grid コンポーネント

Layout / Grid コンポーネントは、画面サイズに応じて要素の配置を自動調整するため、さまざまなデバイスで適切に表示されるレスポンシブ UI を実現できます。

Q12

正解：B

Web スクリーンの主な役割は、データとユーザーインターフェースをユーザーに提示し、操作（インタラクション）を可能にすることです。データの保存やロジックの実行自体は Entity や Action の役割です。

Q13

Web Block は再利用可能な UI コンポーネントを作成するための要素です。複数の画面で同じ UI を共有でき、開発効率と一貫性が向上します。

Q15

クライアント側のフロー内で JavaScript ノードを使うと、独自のスクリプトを実行して高度な UI 制御や動的な動作を実現できます。

Q25

Q36

Q39
