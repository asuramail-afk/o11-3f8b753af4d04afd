Q1

OutSystems でデータベースのリレーションシップ（関係）を定義する場所はどこですか？

  • AAggregate
  • Bプロセスタイマー
  • CWeb Block
  • D画面ロジック
  • Eサーバーアクション
  • Fエンティティデザイナー

正解：F

データモデルのリレーションシップは、エンティティデザイナーで外部キーを設定して定義します（1対1・1対多・多対多）。Aggregate は読み取り用で関係の定義はしません。

Q2

OutSystems で要素をスタイル付けするにはどうしますか？

  • AStyle Class を割り当て、Theme で CSS を定義する
  • BRole を使用する
  • CAggregate の設定を使う
  • DJavaScript を表示する
  • ETimer を呼び出す
  • FEntity を使用する

正解：A

UI のスタイルは、要素に Style Class を割り当て、Theme 内に CSS を定義するのが最も推奨される方法で、一貫した見た目を管理できます。

Q3

入力値の検証ロジックはどこに配置すべきですか？

  • AJavaScript でバリデーションを行う
  • Bサーバーアクションに配置し、保存前に検証する
  • Cタイマーを使って保存後に検証する
  • D送信ボタンだけで検証を行う
  • E保存後に警告を表示する
  • F静的エンティティの検証

正解：B

整合性を守るには、保存の前にサーバーアクション（サーバー側）で検証するのが最も安全です。クライアント側（JavaScript）だけの検証はバイパスされ得ます。

Q4

実行時のパフォーマンスを監視するために使用するツールはどれですか？

  • AService Center の Performance Monitoring タブ
  • BEntity デザイナー
  • CJavaScript デバッグ
  • DWeb Block エディター
  • EAggregate デザイナー
  • FCDT マネージャー

正解：A

Service Center の Performance Monitoring タブで、プロセスの所要時間・画面のロード・ロジックの実行を追跡し、実行時パフォーマンスをリアルタイムに監視できます。

Q5

画面間のナビゲーションを作成するために使用されるコンポーネントは何ですか？

  • EAggregate

Link や Button の Destination プロパティに遷移先の画面を設定することで、クリック操作による画面間のナビゲーションを実現します。
